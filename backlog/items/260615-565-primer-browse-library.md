## Context

`/primer` (pmos-learnkit) produces one verified-source teachable artifact at a time but has
no way to browse the collection, and a pre-built 61-primer PM-curriculum corpus (generated by
the `feat/primer-bundle` harness ‚Äî 61 `*.html` + 61 `*.sources.json` + a rich
`primers-index.json`) has nowhere to live. Both are solved by adopting the released
`/frameworks` browse model: a **committed corpus** under the skill plus an **on-demand,
gitignored, single-file, offline-capable filterable page** built by a zero-dependency
`build-library.mjs`, exposed via a `browse`/`list` verb.

The corpus is transplanted from `feat/primer-bundle` (never merged ‚Äî it carries unrelated
harness contamination) via a path-scoped `git checkout feat/primer-bundle -- <data path>` at
build time. Key design decision: **one unified `library.html`** renders both the curated
corpus (`Collection=Curated`) and any user-generated primers found in `{docs_path}/primer/`
(`Collection=Yours`), retiring the bespoke Phase-5 `primers.html` generator.

Single in-scope skill (`/primer`) ‚Üí one fused story (`260615-dhk`); the corpus, the generator
script, the SKILL.md verb, and the tests ship or fail together, so the G3 one-story-per-skill
default holds.

## Acceptance Criteria

- AC1 ‚Äî The 61-primer corpus (61 `*.html` + 61 `*.sources.json` + `primers-index.json`) is committed under `plugins/pmos-learnkit/skills/primer/data/`; the index parses as a 61-entry array.
- AC2 ‚Äî `scripts/build-library.mjs` emits a single self-contained `library.html` that opens offline from `file://` with zero external requests; `--selftest` passes.
- AC3 ‚Äî The page lists all 61 curated primers as cards (title, category, audience, depth, source-count, word-count, date); each links to its primer HTML, which opens from `file://`.
- AC4 ‚Äî Facet filters (Collection, super_category, category, audience, depth) + free-text search work client-side; "Clear all" resets.
- AC5 ‚Äî User-generated primers in `{docs_path}/primer/` appear under `Collection=Yours`; the shipped corpus under `Curated`; both reachable from one page.
- AC6 ‚Äî `/primer browse`, `/primer list`, and bare `/primer` (no topic) build + open `library.html`; `argument-hint` + a `#browse` phase document it.
- AC7 ‚Äî Phase-5 step-8 listing regen now calls `build-library.mjs` (bespoke `primers.html` generator retired); a freshly written primer appears in `library.html`.
- AC8 ‚Äî With no browser / no Playwright the page is still written and its path printed; opening is best-effort and never fails the run.
- AC9 ‚Äî Tests under `skills/primer/tests/` cover `build-library` against the shipped index; `library.html` is gitignored; skill-eval `[D]` floor + the four lints pass.
