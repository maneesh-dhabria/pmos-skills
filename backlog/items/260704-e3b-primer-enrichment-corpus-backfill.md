## Context

The data-change half of epic 260704-dgq: run the 260704-6rq enrichment engine over all 61 bundled corpus
primers and commit the enriched artifacts. Grounded in `02_design.html` §5 (backfill), decisions D5 (fully
unattended, rubric-gated) / D4 (cap) / D7 (additive), invariants INV-2/INV-4/INV-5/INV-6.

**Depends on 260704-6rq** for the engine (`enrich-references.mjs`) and **260704-rgt** for the `## References`
section already present on every corpus primer (D9 claim-time merge brings both branches into this worktree
before build). Fetch-verifies every added source this run (INV-2) — this story **is** a network run (unlike
rgt's backfill), because enrichment adds new sources that must be verified.

## Acceptance Criteria

- [x] **AC1 — backfill CLI.** New `plugins/pmos-learnkit/skills/primer/scripts/backfill-enrichment.mjs` (or an
  `--all` mode of `enrich-references.mjs`) walks `data/primers/*.html`, applies the 260704-6rq engine to each
  (imported, not re-implemented — INV-1), and atomically rewrites each enriched `.html` + `.sources.json` in
  place. A primer whose every topic is coverage-gate-skipped is a logged no-op, never errored-out.
- [x] **AC2 — fully unattended, rubric-gated (D5).** The run is lights-out: no per-primer human approval. Each
  primer is gated by the engine's rubric check (R1/R11/trust); a primer failing after enrichment is **reverted**
  (INV-6) and recorded, not shipped.
- [x] **AC3 — corpus coverage + run report.** After the run, emit a report (per primer: sources considered /
  verified / added / topics skipped / reverted, plus totals). All 61 primers are either enriched or explicitly
  reported as no-op / reverted — none silently unchanged without a report line.
- [x] **AC4 — additive & convergent (INV-4/INV-5).** Existing prose/sources are preserved; only additions land.
  A second run over the enriched corpus adds no already-present source and does not re-weave (dedup by verbatim
  `url`) — verified by a re-run producing no new source additions.
- [x] **AC5 — corpus committed.** The modified `data/primers/*.html` + `*.sources.json` are committed as the
  enrichment data change; `sources_count` in `primers-index.json` grows honestly with the added sources (or is
  regenerated by the existing index step); `word_count` excludes References (rgt D8). The gitignored
  regenerable `library.html` is unaffected.
- [x] **AC6 — release-prereq scope (§G).** No version-bump / changelog / README / manifest / learnings tasks in
  any build wave — `/complete-dev` (Loop 3) owns those.
- [x] Conforms to `skill-patterns.md §A–§L`; `skill-eval` (`[D]`+`[J]`) passes (skill body unchanged by this
  story beyond the engine already shipped in 260704-6rq — the guard is that the new backfill run + corpus edits
  don't regress the rubric); 4 hygiene lints + `audit-recommended` green.

**Status: done — PASS.** Built unattended (Loop-2 build, `build:59524b3c…`, route:skill inner pipeline). Impl on `feat/260704-e3b` (commit `cfc05a30`: backfill script + 2 test files + 2 engine bug-fixes + SKILL.md doc + 120 enriched corpus files + index refresh; claim `5563906d`), left unmerged for Loop-3 `/complete-dev --epic 260704-dgq`.

Deliverables: `scripts/backfill-enrichment.mjs` — lights-out batch runner that **imports** `enrichPrimer` (INV-1, never re-implemented) and injects a Node-native `verify`: a reachability re-check (D2 link-rot guard, browser-UA GET + Range, 10s timeout, url-cached across the corpus), heuristic trust tier (domain/`source_type` → T1–T4), paywall flag, grounded `summary` as takeaway — deliberately weaker than an interactive `/primer enrich` LLM pass-bar (the corpus was already hard-gated at harvest; documented in the run report + SKILL.md). Refreshes only `sources_count` in `primers-index.json` (pure function → convergent); `word_count` untouched (References excluded, D8); `library.html` never touched.

**Full run:** 61 primers → **60 enriched (+300 curated sources, 1006→1306), 1 coverage no-op, 0 reverted, 0 errors**; 111 URLs fetched, 8 dropped unreachable. Convergence proven: a second run adds 0 and leaves the corpus byte-identical (61 no-op); index refresh is also convergent (0 entries changed on re-run).

**Live run caught + fixed 2 real engine bugs** in `enrich-references.mjs` (the 6rq engine, riding this branch — both were invisible to 6rq's fixture tests, which used clean URLs and single-run fixtures):
1. **Entity encoding** — a source URL with `&` query params is entity-encoded to `&amp;` when woven into HTML, so R1 (body-href ∈ sources.json) and R11 (source ∈ References) compared raw-vs-encoded and **spuriously reverted 24 primers**. `deterministicRubric` now entity-normalises both sides before membership.
2. **Convergence** — the cap was per-**run**, so a re-run accreted the *next* batch of candidates every time (never convergent, violating INV-5's strong sense). Cap is now a per-**primer total** counted from the woven `<p class="primer-enriched">` paragraphs; a primer at the cap is a byte-identical no-op.

Gates: [D] skill-eval-check exit 0 (all checks pass); [J] skill-eval — SKILL.md body change is a single well-grounded `#enrich` paragraph (cites D8/INV-5, documents the batch runner), no regression vs 6rq's **46/47 gated pass** (floor 43); 36 `node --test` cases green (+2 engine: convergence ceiling, entity normalisation; +new `backfill-enrichment.test.mjs`: reachability verify + cache, tier heuristic, paywall, discovery); 4 hygiene lints + `audit-recommended` green.
