## Context

A new pmos-toolkit skill, **`/wiki`**, that lets a PM ramping onto a new charter point at their own
working docs (Notion / Google Docs / GitHub / Figma / Metabase / … — any source with an MCP),
incrementally ingest them through a **resumable, smallest-first, checkpointed pipeline**, and browse
the result as one rich, faceted, skimmable, annotatable **single-file HTML wiki** — filterable to one
workstream or across all of them — with grounded, cited Q&A. The internal productivity tool that turns
a pile of one's own docs into a trustworthy, current mental map of a workstream.

Reverse-engineered from two prior builds in `porter/llm-wiki-prototype` (the FastAPI web prototype,
D1–D26; the skill-native `kb/` four-skill system, D1–D15 — the architectural ancestor) and shaped via
`/shape` (8 user-confirmed forks) into `docs/design-briefs/2026-06-24-wiki-skill.md`, adopted verbatim as
the basis of `02_design.html` and completed during this define run (grill 2026-06-24, 4 branches).

**The two-layer spine (D6).** The atomic unit is the **document** — a byte-exact verbatim mirror plus a
deterministic structured sidecar, the only durable ground truth and the resume/skim unit. The
**understanding layer** (topic pages, onboarding primer, glossary, entity/who-owns-what index, Q&A) is
*derived from documents with citations* and re-generated incrementally on change — so it can never drift
from or contradict the docs. Freshness is "re-derive," not "re-sync two stores."

**The missing primitive.** Nothing in the toolkit ingests a *personal* corpus into a browsable wiki:
`product-context` is hand-authored; `research`/`topic-research` gathers verified *public* sources;
`/frameworks`/`/primer browse`/`/learn-list` browse curated catalog *cards*, not ingested document
*bodies* with section summaries + citations. `/wiki` fills that gap.

## Grill resolutions (define run 2026-06-24 — §10 of the brief)

- **D14 — fork the viewer.** Purpose-build a wiki-only single-file HTML viewer bundled with the skill;
  do **not** extend `library-viewer` (it serves catalog cards, not document bodies). No cross-plugin
  `_shared` dependency, no `/frameworks`/`/primer`/`/learn-list` regression risk.
- **D15 — generic MCP adapter, no per-tool code.** The skill body discovers the right MCP tools for a
  page's source at run time against a generic `fetch / search / extract-links` contract, prompts auth on
  missing MCP (in Claude Code), and surfaces an absent MCP honestly. No source-specific adapter code ships.
- **D16 — workstream inference** = hub-provenance (primary) + corpus tag-overlap (secondary); low-confidence
  parks as `uncategorized`; edited via the `curate` verb.
- **D17 — Q&A retrieval** = ripgrep + BM25 over the structured sidecars, heading-path citation anchors
  (block-ID fallback), scoped by workstream or `--all`.
- **Verb set** = five verbs; `add` (ingest new) ≠ `sync` (re-crawl drifted).
