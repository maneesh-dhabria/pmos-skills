## Acceptance Criteria

(Drawn from epic `260613-yyj` — the change-set for this story.)

- [ ] AC3 — The gate is non-skippable for T2/3 (mirrors the existing non-skippable browser-evidence gate); T1 with no dogfood task is a clean pass (gate N/A).
- [ ] AC4 — `/verify` cites `_shared/dogfooding.md` for the contract and states only the verify-side delta (D9 §K) — no duplicated dogfood-protocol prose.

- [ ] AC7 — **Dogfood (this story's own load-bearing dogfood):** construct a fixture feature folder containing a TN−1 dogfood task with (a) a satisfied verdict and (b) an unsatisfied verdict + accepted residual; run `/verify` against each and confirm the gate yields bare `PASS` for (a) and `PASS-WITH-GAPS` with enumerated gaps + the loud `KNOWN / accepted` line for (b).
