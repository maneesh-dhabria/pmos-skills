Tasks

T1 — Frontmatter + derivation module (TDD)

Add scripts/serve-web-lib.mjs (pure, no I/O server) exporting: parseItems(dir) (read backlog/items/*.md, parse FM per schema.md, skip malformed with warning — D7), buildModel(items, claimsDir) producing the /api/backlog JSON (epics+stories, rollups, queues, facets) per design #api-shape + D5 derivation rules. Tests first (tests/serve-web.test.mjs): fixture backlog → assert epic rollup done/total/blocked, queue membership (groom/next/releases) and readiness match schema.md + #next/#releases rules; malformed item skipped not fatal; empty backlog → empty arrays.

T2 — Zero-dep live server (TDD)

Add scripts/serve-web.mjs: node:http server on 127.0.0.1 ephemeral port (or --port), routes GET / → web/viewer.html, GET /api/backlog → buildModel() result (fresh parse each request — live read, D2), 404 else; --no-open test seam; browser auto-open + URL print + Ctrl-C shutdown borrowed from game-launcher conventions. Read-only: no write paths (D3). Tests: drive the server headless (--no-open), assert /api/backlog returns the model, / returns the HTML, unknown path 404s, a write request method is rejected/ignored.

T3 — Single-file viewer

Add web/viewer.html (inline CSS+JS, no CDN — D4) implementing the three views per design #viewer-contract: Tree (epic→story + progress bars, status badges), Queues (groom/next/releases lanes with display-only next commands), persistent filter bar (status/route/plugin from facets), Refresh button (re-fetch), graceful file:// empty-state. No write controls.

T4 — Wire the web verb into SKILL.md

Edit backlog/SKILL.md: add web to the #routing table; add a ## web — read-only viewer {#web} handler section (resolve repo root, assert node, launch scripts/serve-web.mjs, report URL + Ctrl-C, no prompts); add web + --no-open/--port to argument-hint; add a References bullet for scripts/serve-web.mjs + web/viewer.html. Conform to #conformance (§I: web verb literal; server args in body).

T5 — Live dogfood (load-bearing, served-UI)

Start node scripts/serve-web.mjs --no-open against the real repo backlog; drive the served viewer in a browser (Playwright MCP — serve via the running server, not file://) and assert: tree renders epics+stories with correct counts, the Queues view matches /backlog next --json + releases --json for the same state, a filter narrows the set, Refresh re-reads after an on-disk add. Capture and fix any live-only bug (cap-2 iterate). This is the dogfood gate for a served-UI skill.
