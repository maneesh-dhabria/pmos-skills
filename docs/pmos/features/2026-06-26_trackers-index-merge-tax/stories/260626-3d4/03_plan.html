Executable plan is tasks.yaml; this is the human-readable companion.

Scope

The JS-backed tracker. Remove the committed ~/.pmos/tasks/INDEX.md, turning lib.js:regenerateIndex() into a write-free renderIndex(). The catch (D6): regenerateIndex also performs the workstream→project migration, so the write cannot simply be deleted — the migration relocates to load-time normalization in loadAllItems and must still run on every read. Bare /mytasks becomes web-default with an inline derived fallback. Depends on 260626-psr so the rewritten substrate §5 is in this worktree at claim time.

Risks & decisions

  • Migration regression is the live risk (D6). Deleting regenerateIndex naively would silently drop the workstream→project fold. Mitigation: relocate it to loadAllItems (runs on every read) and add a test asserting it still fires.
  • Byte-equivalence. renderIndex must emit the same buckets the old function did — proven by retargeting testIndex to the returned string rather than reading a file.
  • Dep ordering. Claims after 260626-psr merges so the rewritten §5 (INV-1/2/3) is in this worktree.
