Task waves

Wave 1: study + analyse the live tag distribution (T1). Wave 2: author the vocabulary + synonym data files (T2). Wave 3: wire the normalize step into the importer + gate unknowns (T3). Wave 4: regenerate + validate viewers/prefilter (T4) + skill conformance (T5). Runs after af6's cleaned corpus exists so re-summarized records get tagged once under the new vocabulary (D4).

Key risks

  • Over-collapsing loses signal. Mapping too aggressively could merge genuinely-distinct topics. Mitigation: the vocabulary is reviewable data (D2); the synonym map only collapses provable synonyms/plurals/spellings; ambiguous merges are listed for maintainer review, not auto-applied.
  • IDF weights shift (C3). Collapsing 511→~120 tags changes curated-references-match.mjs IDF weights; a now-ubiquitous tag could get weight ≤0. Mitigation: re-check the prefilter after re-tag; confirm matching still returns sane results on sample topics.
  • Silent re-sprawl on next refresh. Mitigation: a non-zero unknown-tag count after normalization is a build-time failure (D7), so the closed set can't quietly grow.

Verification

  • Final distinct-tag count lands ~100–150; every shipped tag is a vocabulary member.
  • Synonym/plural/spelling variants collapsed; brand/company tags dropped or folded.
  • Unknown-tag count after normalization is reported and gated (non-zero ⇒ build fails).
  • T1 PII gate GREEN; /learn-list browse + /primer facet filters reflect the new tag set (live DOM); IDF prefilter re-checked.
  • Touched SKILL.md passes skill-eval.md + 4 lints.
