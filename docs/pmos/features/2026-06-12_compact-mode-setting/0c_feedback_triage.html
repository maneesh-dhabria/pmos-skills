F1 — compact-checkpoint.md: forced Pause flow unnecessary with autocompact

Evidence: The AskUserQuestion in compact-checkpoint.md offers "Pause to /compact, then resume" as an option. When the user has configured an autocompact window in Claude Code, this flow requires unnecessary manual steps: pick Pause → type /compact → type /feature-sdlc --resume. Proposed fix: Add compact_mode: manual | auto to .pmos/settings.yaml (default: manual). When compact_mode: auto, skip the AskUserQuestion entirely at every compact checkpoint; log compact_mode: auto — checkpoint at <phase> skipped; autocompact active and proceed. The manual path is unchanged.
