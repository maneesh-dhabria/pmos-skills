/architecture — deepening pass + hygiene — Spec

tier: 3 · type: feature · feature: architecture-deep-pass · date: 2026-05-14 · status: Ready for Plan · requirements: docs/pmos/features/2026-05-13_architecture-deep-pass/01_requirements.html · target skill: plugins/pmos-toolkit/skills/architecture/ · target platform: generic · acceptance criteria: plugins/pmos-toolkit/skills/feature-sdlc/reference/skill-patterns.md §A–§F

1. Problem Statement

The /architecture skill is a fast, deterministic rule engine that runs L1 universal + L2 stack-specific (TS via dependency-cruiser, Python via ruff) + L3 per-repo overrides and emits JSON to stdout. On a real Typer-heavy backend (book-companion, ~900 LOC service files), v2.48.0 produced 586 findings dominated by 218 print() false positives and 710 info-noise rows, silently skipped monorepo stacks because pyproject.toml lived a level below the scan root, and gave no human-readable artifact to share. This spec defines the technical implementation of a discovery-capable deepening pass + signal-hygiene fixes + cross-module structural detectors + an HTML+MD+JSON triplet output that replaces stdout-JSON, and breaks the severity axis in favour of a disposition (Must Fix / Should Fix / Won't Fix) axis. Primary success metric: total findings on book-companion's backend/ drop from 586 to ≤120 (≥80% reduction); on --deep, ≥3 leaky/shallow classifications with substring-grep-verified evidence.

3. Non-Goals

  • Not turning --deep on by default — opt-in only; mechanical pass remains the deterministic default.
  • Not rewriting /improve-codebase-architecture or absorbing it wholesale; the deep-pass vocabulary is bundled at reference/deepening-vocabulary.md with attribution.
  • Not auto-fanning monorepos without explicit --monorepo; warn-loudly is the default.
  • Not writing new tools in languages other than POSIX bash + Python 3 (already-required prereqs).
  • Not preserving back-compat for severity: keys — the rename to disposition: is breaking; user L3 files parse-error with a one-line rename hint.
  • Not writing JSON to stdout; stdout is empty (D17 below).
  • Not promoting findings to Nygard ADRs; ADR machinery and --no-adr flag are removed.
  • Not double-reporting — size-class findings (U001/U002/PY005/PY006 only) on a deep, reappears file demote to Won't Fix with deep_pass_cleared: true, not silently dropped.
  • Not crashing when no subagent/Read tool is available — runtime probe sets tools_skipped: ["deep_pass:no_subagent_tool_use"] and the run succeeds with mechanical findings only.
  • Not capping deep-pass candidate count — discovery-capable; seed_hint[] is advisory.
  • Not retaining byte-identical determinism — tools/check-determinism.sh is removed; users self-check via --baseline against a prior JSON sidecar.

5. User Personas & Journeys

5.1 Primary persona — Tech lead auditing a backend

Senior engineer / tech lead running /architecture audit . on a real repo before a refactor, during onboarding, or as a periodic check. Concrete example: maintainer of book-companion (~900 LOC Python backend service files). Needs (a) noise gone, (b) prioritized action list, (c) a shareable artifact.

5.2 Journey — Quick local audit on a Typer-heavy backend

  1. Developer runs /architecture audit backend/ at the repo root.
  2. Phase 3 scan detects 1 stack (python via pyproject.toml); idiom detector finds Typer imports + walks AST to compute exempt_ranges[] per file (function-scoped: @app.command-decorated bodies only).
  3. Phase 4 evaluators run L1 grep checks + L2 ruff (PY001–PY008) + new PY009 cycle-py + U011 dup-signature.
  4. Phase 4.5 — --deep absent — skipped.
  5. Phase 5 reconciles idiomatic exemptions; computes module_metrics[]; emits godmodule_candidates[] (top-10) — for the appendix.
  6. Phase 6 emits HTML+MD+JSON triplet to docs/pmos/architecture/2026-05-14_backend.{html,md,json}. Stderr: Wrote docs/pmos/architecture/2026-05-14_backend.html: 4 Must Fix, 31 Should Fix, 85 Won't Fix in 47 files.
  7. Developer opens the HTML, reads the Must Fix section, opens the top file, makes the fix.

5.3 Journey — Monorepo audit

  1. Developer runs /architecture audit . at the monorepo root (no stack markers at root).
  2. Phase 1 depth-2 walk detects backend/pyproject.toml + frontend/package.json; populates monorepo_detected[].
  3. Phase 2 emits stderr warning: No stack markers at scan root; found backend/pyproject.toml, frontend/package.json. Re-run with --monorepo to fan out, or audit each stack directly. Exit 0 with empty findings.
  4. Developer re-runs with /architecture audit . --monorepo; orchestrator iterates each detected stack; emits per-stack triplet (2026-05-14_backend.{html,md,json}, 2026-05-14_frontend.{html,md,json}) + a top-level 2026-05-14_index.{html,json}. Index HTML links each per-stack HTML; index JSON carries stacks: [{name, json_path, html_path}].

5.4 Journey — CI delta mode

  1. CI runs /architecture audit . --since origin/main --baseline docs/pmos/architecture/2026-05-12_root.json on a PR branch.
  2. Phase 3 scan filters to git diff --name-only origin/main...HEAD intersected with normal file-walk results.
  3. Phase 6 loads baseline JSON; diffs against current findings via (rule_id, file, line) tuple identity; emits diff{} block (new[], dropped[], unchanged[] counts + samples).
  4. Stderr: Wrote docs/pmos/architecture/2026-05-14_root.html: 3 Must Fix (new), 0 Should Fix (new), 0 Won't Fix (new); 0 dropped; 583 unchanged on base.
  5. CI step parses the JSON sidecar's diff.new[], fails the PR if any entry has disposition: must_fix.

5.5 Journey — Deepening pass on legacy code

  1. Tech lead runs /architecture audit . --deep before a major refactor.
  2. Mechanical pass (Phases 1–4) completes; Phase 5 produces godmodule_candidates[] → seed_hint[] (top-10).
  3. Phase 4.5 — Deepening pass:
    1. Runtime probe: attempt a no-op subagent dispatch + Read probe. On failure → set deep_pass.skipped_reason: "no_subagent_tool_use"; jump to Phase 5.
    2. Payload build (D15): assemble module_metadata[] for every Python module (path, fanin, fanout, public_symbols, LOC). Soft-warn to stderr at >1,000 modules; hard error at >5,000.
    3. Subagent dispatch: one blocking Task subagent with system prompt = vocab cheatsheet (reference/deepening-vocabulary.md body) + payload (module_metadata[] + seed_hint[]) + the verbatim return-shape template. Subagent has Read + Grep tool access. Per-call timeout 600s; no retry (LLM non-determinism makes retry less valuable; failure → fall back to mechanical).
    4. Validation (D11): for each returned candidate, assert (a) module resolves to a real file; (b) every cross_module_patterns[].evidence verbatim-substring-greps inside the named file; (c) classification ∈ {deep, shallow, leaky}. Any miss → drop the candidate and emit a stderr warning naming the violation. If >50% of candidates fail validation → deep_pass.skipped_reason: "validation_failed"; preserve mechanical findings.
    5. Promotion: leaky → finding with rule_id: DEEP_LEAKY, disposition: must_fix; shallow → rule_id: DEEP_SHALLOW, disposition: should_fix; deep → no finding emitted (the absence of a problem).
    6. Reconciliation: for each deep, deletion_test.outcome: reappears classification, scan existing findings for size-class entries (U001/U002/PY005/PY006) on that file; demote each to disposition: wont_fix with deep_pass_cleared: true. Other rule classes unaffected.
  4. Phase 6 emits HTML; Must Fix section leads with DEEP_LEAKY entries (rationale + proposed_reshape paragraphs); Won't Fix > "Cleared by deep pass" sub-section preserves the demoted size-class rows.

5.6 Error / edge journeys

  • No subagent on --deep: deep_pass.skipped_reason: "no_subagent_tool_use"; mechanical run completes; exit 0.
  • Subagent validation fails (>50%): deep_pass.skipped_reason: "validation_failed" + detail; mechanical preserved; exit 0.
  • --scaffold-l3 with existing file: stderr refusing to overwrite existing <path>; exit 64.
  • Schema-v1 baseline: parse-error with baseline file uses legacy schema_version 1; v1 used `severity`, not `disposition`. Regenerate. Exit 64.
  • L3 with legacy severity:: parse-error at Phase 1: principles.yaml uses legacy 'severity:' key; rename to 'disposition:' (block→must_fix, warn→should_fix, info→wont_fix). Exit 64.
  • Scan root not a git repo: risk_score.churn = 0; --since / --baseline → exit 64 if either passed with scan root is not a git repo; --since/--baseline unavailable.
  • --baseline index.json without --monorepo: stderr index baseline requires --monorepo on current run; aborting; exit 64.
  • Deep-pass module count >5,000: stderr --deep payload exceeds 5,000-module cap; audit a sub-tree or omit --deep; exit 64.
