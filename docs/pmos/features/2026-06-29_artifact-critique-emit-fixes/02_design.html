Epic design — /artifact-critique emit-pathway fixes

Epic 260629-9ne · route: skill · plugin: pmos-toolkit · define / Loop 1 · source: /reflect retro (/artifact-critique, 1 run, 2026-06-29)

Why this epic

A single /artifact-critique run on pov/pov_v6.html delivered fully on its claimed contract — correct 10-axis scorecard, ≥40-char verbatim quotes on every axis, valid embedded findings JSON, published artifact, no pushback on critique content. The retro surfaced four gaps in the emit pathway only: two friction-level (an undocumented platform fallback for the HTML emit, and an undocumented protocol when the deterministic gate can't run) and two nits (an advisory-reviewer line that wasn't always surfaced, and the Phase 0 resolution lines never appearing on a no-stderr harness). None touch the critique engine or the rubric — all four are documentation / contract fixes in the skill's SKILL.md.

Scope & shape

This is a singleton skill epic (D18 wrap): every fix edits the same single file — plugins/pmos-toolkit/skills/artifact-critique/SKILL.md — so the D24 litmus fails any multi-story split. One story (260629-6j0, route: skill). No rubric substrate, no _shared/html-authoring/ change, no script change: the critique-eval.mjs gate and the multi-file emit contract are documented around, not modified.

Decisions (maintainer-approved, this define run)

IDDecisionProvenance
D1 Conditional self-contained HTML fallback; multi-file substrate stays default. When the host platform exposes a first-class Artifact/canvas publish tool, the skill MAY emit a single self-contained HTML (inline <style>/<script>, no external asset links), retaining the embedded pmos-critique-findings block and inlining the comments overlay (comments.js/comments.css) best-effort, and publish it via that tool in the same phase — never requiring the user to initiate publication separately. The multi-file assets/ substrate emit remains the default for --out / pipeline-folder writes. Any overlay capability not inlined is named in ## Limits. F1; grilled via AskUserQuestion (chose "conditional fallback, multi-file stays default" over "self-contained becomes default" / "verbatim minimal").
D2 Node-unavailable protocol for the deterministic gate. When node exec is unavailable, skip critique-eval.mjs, manually validate the deterministic checks by inspection (especially E-quote-in-source — every quote a real substring — plus E-axes-complete and E-applicable-consistency), add a ## Limits entry ("deterministic gate not run (Node unavailable) — findings manually validated"), and proceed. Never block; never silently omit the limitation. Mirrors the /feature-sdlc skill-eval "no bash → LLM-judge fallback" stance. F2; adopted (clear). Strengthened from the verbatim "skip" to "skip + manual-validate" to keep Inv-5 honesty.
D3 Always-surfaced advisory-reviewer line. Phase 7 Tier-2 MUST emit exactly one chat-summary sentence reporting the advisory reviewer's outcome regardless of findings: Tier 2 advisory reviewer: ran <inline|subagent> — <N findings | no findings>. An empty review is no longer silent. F3; adopted (modify — strengthen the existing "surface its notes" contract to always-emit).
D4 No-stderr surfacing of Phase 0 resolution lines. On a harness with no distinct stderr channel, surface the Phase 0 resolution lines (mode: … and output_format: …) inline as a brief code block at Phase 0 entry, then continue. Cross-skill note: this is a repo-wide convention (every pmos skill prints these to stderr); scoped to this skill per the feedback, flagged as a promotion candidate for the shared non-interactive block but NOT promoted here (out of scope for a single-skill epic). F4; adopted (nit). Cross-skill promotion deliberately deferred.
