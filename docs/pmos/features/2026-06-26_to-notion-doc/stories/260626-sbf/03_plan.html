Risks & mitigations

  • Zero-dep HTML parsing is the hardest deterministic piece. Mitigation: parse only the conversion-relevant tag set tolerantly; unmappable structure → ambiguous node (never a crash). The dogfood doc (real HTML with 9 tables + 2 SVGs) is the forcing function.
  • File Upload API needs a real token + live calls — not selftestable offline. Mitigation: mock the HTTP layer in selftest (assert payload shapes + ladder branching); the live path is exercised only in the T9 dogfood, and the stub rung guarantees a working zero-token path.
  • Notion content can't be committed (maintainer's workspace). Mitigation: dogfood evidence is the conversion report + reconciliation table only, under dogfood/.
  • Table full-width is not API-controllable — must not be promised. Mitigation: documented as a known Notion limit in reference/notion-blocks.md; the win is column/row fidelity (AC7).
