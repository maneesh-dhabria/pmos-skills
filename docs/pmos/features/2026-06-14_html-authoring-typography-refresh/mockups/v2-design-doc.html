/backlog web viewer — nomenclature & conceptual clarity

Bottom line

The web viewer dumps the serve-web-lib data model straight onto the page with its internal vocabulary and no translation layer — so a reader sees @null, raw lifecycle states, and unlabelled counts. This epic adds the missing plain-language layer without touching the server contract.

Problem & findings

A /design-crit pass on the viewer surfaced seven findings, all rooted in one cause: the viewer renders the model's field names rather than translating them into reader-facing language. None require a server change.

Scope guard

Three files change — viewer.html, serve-web-lib.mjs, and the test — and nothing in the server contract or _shared/. The fix is canonical with a defensive viewer-side guard, so no other consumer regresses.

The null-coercion fix

The leak happens because parseScalar returns the sentinel string when a field is unset. Coercing at the parser is the canonical fix; the viewer guard is defence-in-depth.

// serve-web-lib.mjs — canonical fix
function parseScalar(raw) {
  if (raw == null || raw === '@null') return '';   // coerce sentinel → empty
  return String(raw).trim();
}

// viewer.html — defensive guard
const cell = parseScalar(item[field]) || '—';
