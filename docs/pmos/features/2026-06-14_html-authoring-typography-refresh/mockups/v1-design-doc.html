/backlog web viewer — nomenclature & conceptual clarity

The web viewer dumps the serve-web-lib data model straight onto the page with its internal vocabulary and no translation layer — so a reader sees @null, raw lifecycle states, and unlabelled counts. This epic adds the missing plain-language layer without touching the server contract.

Problem & findings

A /design-crit pass on the viewer surfaced seven findings, all rooted in one cause: the viewer renders the model's field names rather than translating them into reader-facing language. None require a server change.

#FindingSeverity
F1Column headers use internal field names (kind, route) with no subtitles.Should-fix
F2Releases queue shows items with 0 done stories; no progress indicator.Should-fix
F3Lifecycle status filter is a flat list of raw states, not grouped.Nit
F4EPIC vs story rows are visually indistinguishable.Blocker
F5@null leaks into the UI when a scalar field is unset.Blocker
F6No legend or tooltips explaining the queue semantics.Nit

Decisions

Each finding maps to a locked decision and one or more functional requirements. All seven were accepted as Recommended at grooming.

Scope guard

Three files change — viewer.html, serve-web-lib.mjs, and the test — and nothing in the server contract or _shared/. The @null fix is canonical (in parseScalar) with a defensive viewer-side guard, so no other consumer regresses.

Functional requirements

  1. FR-1 — Render reader-facing column labels from a label map, not raw field keys.
  2. FR-2 — Each column header carries a muted one-line subtitle describing what it means.
  3. FR-3 — The Releases queue filters out items with zero done stories and renders a progress bar.
  4. FR-6 — parseScalar coerces an unset scalar to an empty string, never the literal @null.

The null-coercion fix

The leak happens because parseScalar returns the sentinel string when a field is unset. Coercing at the parser is the canonical fix; the viewer guard is defence-in-depth.

// serve-web-lib.mjs — canonical fix
function parseScalar(raw) {
  if (raw == null || raw === '@null') return '';   // coerce sentinel → empty
  return String(raw).trim();
}

// viewer.html — defensive guard
const cell = parseScalar(item[field]) || '—';

Stories

One fused story — 260614-8v4 — carries all seven findings. Task T6 is a required Playwright live-dogfood that drives the served viewer and asserts no @null renders and the pills appear.
