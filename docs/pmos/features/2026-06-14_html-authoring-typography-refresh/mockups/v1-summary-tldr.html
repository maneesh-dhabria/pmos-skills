Porter Metric Store POV

Porter's metric stack fails in five concrete ways — and the Metric Store fixes all five by making the semantic layer the durable substrate: analysts author each metric once, anyone asks for it in plain English, and the system returns the right number or an honest confidence band — never a silent guess.

Bottom line up front

Porter's metric stack fails in five concrete ways: high friction to access numbers, no proactive signal on change, poor cross-team discoverability, "Daily Active Partners" meaning three different things across three dashboards, and AI tools that invent SQL and return plausible-but-wrong answers. The Metric Store (Project Argus) fixes this by making the semantic layer the durable substrate — analysts author each metric once in Git-versioned YAML, and anyone asks for it in plain English via a single skill on Claude Code, Claude Cowork, or Claude AI, getting the right number or an honest confidence band, never a silent guess.

What it decides

  • Direction: build a skills layer on top of MetricFlow and existing catalog MCPs — not Snowflake Cortex Analyst (same authoring effort, plus vendor lock on the AI surface), not a SaaS semantic layer (recurring license for capability Porter already runs), not an AI wrap over Metabase tiles (scales existing definition drift instead of fixing it).
  • V1 cut: Claude Code + Cowork + Claude AI, 4–5 pilot charters selected on three hard criteria: a metric-fluent PM co-owner, an exec sponsor, and a declared commitment to make /query-metrics the charter's default for ad-hoc metric questions. Slack deferred to Next.

Three confidence tiers

The core product contract. Every answer is labelled with the tier it came from, so a consumer always knows how much to trust it.

TierWhat it coversConfidence
Defined Pre-authored metrics; same question, same answer every time. Covers DAP, Revenue, Orders and the metrics each charter names at kickoff. 100%
Variant Standard math on defined metrics with the same data — MoM growth of Revenue, p10 when p90 is defined. Compiled deterministically by MetricFlow. 100%
Undefined Composed on the fly from semantic-model measures; confidence score and composition shown inline; per-domain kill switch fires automatically. ≥70% / domain
The honest part

parse_metrics_request() is the only AI step in the consumption path. It parses intent, routes via the Catalog MCP, asks one clarification if ambiguous, and never guesses silently — the difference between a tool you can cite in a board deck and one you can't.
