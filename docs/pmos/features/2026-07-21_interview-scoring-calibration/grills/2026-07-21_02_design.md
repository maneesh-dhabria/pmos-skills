## Resolved

- **Q1 — Does `sak` backfill the 8 bundled archetypes, or only emit descriptors on newly-authored sheets?**
  → **Third story, `sak → 1a4 → jb6`.** The corpus carries 47 dimensions and **zero** descriptors, so without backfill the epic's headline fix is dormant on every default path. But 188 descriptors is authored judgement content — a full `/execute` run — and fusing it into `sak` would make one `skill-eval` span "author the anchor contract" *and* "make 188 calibration judgements". → **D6**.

- **Q2 — What does tagging a dimension "untested" (D2d) do to the arithmetic?**
  → **Exclude from the denominator + renormalize + coverage gate.** Untested-weight-% becomes a first-class reported figure; crossing **30%** forces either an *insufficient evidence* reco or a non-empty `data-reco-rationale`. Score-neutral was rejected as an INV-4 violation (it invents a score for evidence that does not exist). → **D7**.

- **Q3 — A provided guide with no rubric: what is the bar?** *(maintainer-raised mid-grill)*
  → **Synthesize → write inline → agree → score.** Descriptors are derived per dimension, **written into the sheet as real `data-level` anchors** (self-describing sheet; the next run inherits the same bar), presented for agreement, and only then used. Blocking. This **retires** the design's original "absent anchor → byte-identical prior behavior" degrade path — which was the bar-inflation hole itself. → **D8**, and INV-1 was rewritten around it.

- **Q4 — Headless disposition of the new agreement gate.**
  → **AUTO-PICK the synthesized rubric.** ⚠️ **Maintainer override** — the Recommended option was DEFER + score provisionally + stamp draft, mirroring the skill's existing `<!-- defer-only: free-form -->` duration-confirm precedent. The maintainer traded the guarantee for unattended throughput. Mitigation added that the choice does not preclude: the sheet stamps `data-rubric-provenance="synthesized-auto"`, so a reader can always tell the bar was never human-agreed. Logged under *Accepted risks* in the design.

- **Q5 — Where does the per-dimension evidence sweep render?**
  → **Collapsed `<details data-card="evidence-sweep">` inside `filled-scorecard.html`.** Preserves the self-contained-single-artifact property, keeps the gate checkable, and costs the panel nothing until someone audits a score. → **D9**.

- **Q6 — Does the calibration gate apply to output (b), `interviewer-notes.html`?**
  → **Method yes, gate no.** (b) has *no* scoring anchors at all — no `data-dim`, no `data-v`, no `data-selected` — so the script structurally cannot run against it. But (b) makes below-bar claims from the same transcript with the same reading-order risk, so sweep-then-conclude and the adversarial below-bar pass apply to Phase Coach as method. Giving (b) its own scoring surface is explicitly deferred. → **D11**.

- **Q7 — Does the 2-story split survive?**
  → **No — split to 4.** `jb6` was carrying 5 FRs, a new gate script, four new anchors, a completeness gate *and* a stamp change. The completeness gate (F2) and the citation stamp (F3) touch different files, have zero dependency on the calibration work, and are the retro's *friction* and *nit* findings rather than its blocker. They move to **`z5n`** (no deps, buildable immediately). Combined with D6, the epic is `sak · 1a4 · jb6 · z5n`.

- **Q8 — Is INV-2's original "record a verdict, gate its presence, leave correctness advisory" genuinely §H-legal?**
  → **No — it was a hard gate wearing a presence costume.** A model satisfies "stamp `consistent`" on every dimension without reading a word. Replaced with `data-note-matches-level="<n>"` — the model records *which level its prose actually describes*, and the **script** compares that integer to `data-selected`. The judgement stays with the model; the check becomes a mechanical integer compare the script owns. Textbook §H, and it catches the retro's exact tell. → **D10**.

- **Q9 — What happens on a note-vs-score mismatch?**
  → **Require a non-empty `data-score-rationale`, not a hard fail.** A mismatch is sometimes legitimate (a red flag drags the number below what the prose alone implies). Hard-failing every mismatch creates a direct incentive to rewrite the note to match the number — which destroys the signal the gate exists to read. Rationale-on-disagreement is the same shape as the reco-vs-modal gate: one pattern, used twice. → **D10** (second half).

## Gaps surfaced

- **⚠ Latent §H violation, live in `main` today.** `fill-scorecard.mjs` parses `data-weight` into `{id, weight, scale, …}` and **never computes anything with it** — there is no weighted-score routine anywhere in the skill. The retro's "2.10 → 3.00" was arithmetic the model performed in its head, which §H forbids outright. D3's "the script computes the modal and weighted score" is therefore a **new capability closing an existing bug**, not a re-homing. This materially changes FR-6's plan and is flagged in the design's *Why this epic*.

- **⚠ Per-seniority descriptor variants — unresolved, blocks `1a4`'s authoring.** `level-ladder.md` already varies competency *weights* across `apm · pm · senior-pm · group-pm · director · vp`. "Solid, some prompting OK" plainly means something different for an APM than for a Director. If descriptors must vary by seniority, `1a4` goes from **188 to ~1,128 descriptors** and needs re-scoping (or a parameterized descriptor with a seniority-sensitive clause). Must be settled in `1a4`'s `/plan` before a single descriptor is authored. Recorded under *Open questions* in the design.

- **The 30% untested-coverage threshold (D7) is a proposal, not a calibration.** No real round has been measured against it. Ship it as one named constant in `check-scoring-calibration.mjs` so it is trivially tunable, and revisit after first use.

- **Residual risk from the Q4 override.** Headless runs can now score against a bar no human ever saw. Named and accepted in the design's *Accepted risks*; auditable after the fact via `data-rubric-provenance`.

- **`reference/output-shapes.md:32` mentions a `.calib` good/average/poor line, but it is unanchored prose** — there is no corresponding markup in `reference-skeleton.html`. It was considered as a bar source and rejected in triage; noted here because it is a live inconsistency in `/interview-guide`'s own reference doc, orthogonal to this epic.
