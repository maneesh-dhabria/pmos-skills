The two findings

F2 [friction] — a promised section shipped empty

The "Observer's independent read" block shipped with an unfilled [ … your observations go here ] placeholder while the run reported complete, across all three passes — and the observer had explicitly asked to share their observations. An empty promised section reads as "nothing to say", which is the opposite of the truth.

F3 [nit] — a hand-maintained count drifted

The proof-of-pass comment is written by the model today, so it went stale on every correction and was hand-patched 19 → 22 → 24:

<!-- citations verified: <N> transcript-tier, <M> notes-tier[, <K> submission-tier], <YYYY-MM-DD> -->

§K one fact, one home: the count cannot drift if no human ever types it. The gate already computes these numbers — it just prints them instead of writing them.

Risks & mitigations

  • Stamping on a failing gate. The worst possible outcome: a proof-of-pass comment written when the gate did not pass, which is worse than no comment because it is affirmatively misleading. --stamp writes only on exit 0; T6's third fixture is exactly this case.
  • Non-idempotent stamping. A naive append produces two comments after the second run and resurrects the drift the fix exists to kill. T2 replaces in place; T6 asserts a byte-identical file on a second run over unchanged input.
  • Regressing the citation gate. The half of the contract that worked (INV-3). Only who writes the comment moves — not the tier model, not the ≥40-char verbatim rule, not the STOP behaviour, not the counts themselves. T6 asserts the existing tests still pass.
  • The completeness scan fabricating content. INV-4. The gate captures or stamps; it never writes the observer's read for them. T5 states this explicitly and offers no "generate it" path.
  • False positives on legitimate prose. Bracketed text appears in normal writing. T4's detection must key on the skeletons' actual placeholder conventions (pinned in T1), not on a loose bracket regex — a false positive would stamp a complete artifact "draft".
  • Flag creep. --stamp is a script argument, not skill CLI surface. argument-hint stays unchanged (§I / INV-7).
