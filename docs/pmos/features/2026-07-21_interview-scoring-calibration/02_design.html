Epic design — interview scoring calibration

Epic 260721-k1x · route: skill · plugin: pmos-managerkit · define / Loop 1 · source: /reflect retro (/interview-feedback, 1 run, 2026-07-21) · grilled --depth deep (9 branches)

Why this epic

One /interview-feedback run split cleanly down the middle of the skill's own claimed contract. The grounding half held — inputs resolved, the Gemini transcript was treated as tier 1, both artifacts shipped with verbatim-verified citations, and check-citations.mjs exited 0 on every pass. The scoring half failed twice, and only user pushback caught it: the first challenge ("only initial probed answers, or did he eventually get there?") moved Dim 1 and flipped the recommendation off No-Hire; the second ("are you sure about the others — look at the entire transcript") moved Dim 2/3/4/6 and flipped it again to Hire. The weighted score travelled 2.10 → 3.00 and the recommendation swung two full bands, on the same evidence, with no new input.

The through-line is a gap between two different claims. The citation gate proves a quote is real. Nothing proves the score follows from the evidence at the rubric's own bar. That second surface is entirely unguarded, and it failed in two distinct, separately-fixable ways:

  • Time-bias — the run scored the first probed answer per dimension and discounted where the candidate eventually landed. Scoring happened linearly, while reading; an eventual insight cannot be credited if it has not been collected yet. This is a reading-order defect.
  • Bar-inflation — the run applied a senior-signal "did he lead with it, unprompted, up front?" standard as the pass line, when the sheet's own level-3 reads "solid, some prompting OK". The level-4 ceiling was borrowed down to level 3, so every gap scored a 2. This is a calibration defect.

The bar has no machine home

Triage surfaced the fact that reshapes the whole fix. The strongest proposed safeguard — "paste the scorecard's own level-3 text and score against that wording" — has nothing to read. The bundled scorecard-skeleton.html scale is bare numbers, and so is every archetype sheet that derives from it:

<div class="scale" data-scale="1-4">
  <span data-v="1">1</span><span data-v="2">2</span><span data-v="3">3</span><span data-v="4">4</span>
</div>

Measured across the bundled corpus: 8 archetypes · 47 dimensions · 0 level descriptors. The level-3 wording the failing run was measured against came from a hand-authored sheet, not from anything the skill guarantees. /interview-guide authors the sheets and already emits data-duration and data-budget; the bar belongs in the same place. So this epic spans the skill pair, exactly as the duration/budget interop did (epic 260707-rbc → 260709-qfn).

Latent §H violation, live today (found while grilling). fill-scorecard.mjs parses data-weight and never computes anything with it — there is no weighted-score routine anywhere in the skill. The retro's "2.10 → 3.00" was arithmetic the model performed in its head, which §H forbids outright. FR-6's script is therefore a new capability closing an existing bug, not a re-homing of an existing calculation. Plan it as such.

Decisions (maintainer-approved, this define run)

D1–D5 were settled in triage; D6–D11 in the --depth deep grill.

IDDecisionProvenance
D1 The bar becomes a machine anchor: data-level on each scale option, authored by /interview-guide. Each <span data-v="n"> in a data-scale container gains data-level="<descriptor>" carrying that level's at-bar wording for that dimension. Gated all-or-none per dimension (a partially-described scale is exactly the ambiguity that invites borrowing the ceiling down), each descriptor non-empty. Mirrors the data-duration/data-budget precedent: authored once in the sheet — the single machine home — read, never re-derived. F1 [blocker], safeguard 2; grilled (chose new anchors over "read whatever the sheet carries" / "consume the reference's .calib prose").
D2 Phase Score's scoring method is rewritten: sweep-then-score, quote-the-bar, floor/ceiling split, untested-not-failed. (a) Reading order: sweep the whole transcript per dimension and collect every instance — early and late, prompted and unprompted, timestamped — before assigning any number. (b) Quote the bar: quote the dimension's own data-level text for the level under consideration and score against that wording; "unprompted / up-front" language belongs to the ceiling and may not be borrowed down to the pass line. (c) Floor/ceiling split: "did they demonstrate it at all, at any point?" sets the floor; "how much nudging did it take?" caps the ceiling (heavy → 2, one pointed nudge → 3, none → 4). Prompting adjusts the ceiling; it never zeroes out a genuine demonstration. (d) Untested ≠ failed: a competency never probed is tagged untested, not scored below bar. F1 [blocker], safeguards 1–4; adopted.
D3 Four calibration gates, §H-decomposed, enforced by a new scripts/check-scoring-calibration.mjs as a second blocking STOP-before-done gate. Each is decomposed into a deterministic condition the script owns:
  • Evidence-sweep — every scored data-dim carries a data-card="evidence-sweep" block with ≥1 timestamped instance, present before a data-selected score exists. Presence + ordering.
  • Adversarial below-bar — every dimension scored below its at-bar level carries a non-empty data-rebuttal answering "what is the strongest case this is actually at bar?". Conditional presence.
  • Note-vs-score — see D10: the model records which level its prose describes; the script compares that integer to the assigned score. Integer comparison, not judgment.
  • Reco-vs-modal — the script computes the modal and weighted score (arithmetic → script, never the model) and, on band disagreement, requires a non-empty data-reco-rationale. Arithmetic + conditional presence.
Non-zero exit is a hard STOP: the scorecard is not presented and the run does not declare done. This gate joins the citation gate; it does not replace it.		 F1 [blocker], safeguards 1, 5, 6, 7; grilled (chose the four-gate set over "all 7 hard" / "one combined checklist gate").
D4 Completeness gate + draft stamp: an unfilled promised section blocks "done". Before either artifact is declared complete, scan for unfilled placeholders and empty promised slots (bracketed placeholder prose, empty data-input slots, un-substituted {{…}} tokens). Interactive → prompt to capture the content in the same run (the observer's independent read is the motivating case). Declined, or --non-interactive → stamp draft — pending <slot>. Never silently passes an artifact with a hole in it; never fabricates the content to fill one. F2 [friction]; grilled (chose gate + stamp over "stamp only" / "prompt only").
D5 check-citations.mjs --stamp owns the proof-of-pass comment. On exit 0 with --stamp it writes/patches <!-- citations verified: … --> into the HTML itself (idempotent replace). SKILL.md stops instructing the model to hand-write it. §K one-fact-one-home: the count cannot drift because no human ever types it. The hand-patched 19 → 22 → 24 sequence becomes structurally impossible. F3 [nit]; grilled (chose script-writes-it over "countless marker" / "both").
D6 Backfilling the 8 bundled archetypes is its own story (1a4). The corpus carries 47 dimensions and zero descriptors, so without backfill the epic's central fix never fires on any default path. But 188 descriptors of authored judgement content is a full /execute run, and fusing it into sak would make one skill-eval span "author the anchor contract" and "make 188 separate calibration judgements". Separate story, separate reviewable diff, depends on sak for the contract. Grill Q1 (chose third story over fold-into-sak / defer / partial 4–5 archetypes).
D7 Untested dimensions leave the denominator; coverage becomes a gated figure. A dimension tagged untested (D2d) is excluded from the weighted score and the remaining weights renormalize. check-scoring-calibration.mjs reports untested-weight-% as a first-class coverage figure. When it crosses 30% (one named constant), the recommendation must either be insufficient evidence or carry a non-empty data-reco-rationale defending a call made on partial coverage. All arithmetic is the script's (§H). This closes the hole where a round that probed 3 of 6 dimensions silently produces a confident number. Grill Q2 (chose exclude+renormalize+gate over report-only / always-insufficient / score-neutral — the last violating INV-4).
D8 A provided guide with no rubric is materialized and agreed before any scoring. When the resolved sheet carries no data-level descriptors, /interview-feedback derives them per dimension from the guide's own strong/weak markers, its .calib line, the archetype corpus and the seniority; writes them into the sheet as real data-level anchors (the sheet becomes self-describing, and the next run inherits the same bar); presents them for agreement; and only then scores. Blocking — no dimension is scored against an unagreed bar. This retires the original "absent anchor → score uncalibrated" degrade path, which was the bar-inflation hole itself. Under --non-interactive the agreement gate AUTO-PICKs the synthesized rubric (maintainer's explicit call, overriding the DEFER+provisional recommendation); the sheet records data-rubric-provenance="synthesized-auto" so a reader can always tell the bar was never human-agreed. Residual risk is accepted and named in Accepted risks. Maintainer-raised mid-grill; shape grilled (chose synthesize+write+confirm over confirm-without-persisting / refuse / route-back-through-/interview-guide). Headless disposition is a maintainer override.
D9 The evidence sweep renders as a collapsed <details> inside the scorecard. One <details data-card="evidence-sweep"> per dimension, collapsed by default, under that dimension's notes. The panel reads scores and notes exactly as today; anyone auditing a score expands one block and sees every instance early/late/prompted/unprompted with timestamps. Preserves the standing self-contained-single-artifact property, keeps the gate checkable, costs the reader nothing until they want it. Grill Q5 (chose collapsed-in-scorecard over separate working artifact / always-visible).
D10 Note-vs-score is adjudicated by integer comparison, not by a self-reported verdict. Each dimension records data-note-matches-level="<n>" — the level whose descriptor the written note actually describes — and the script compares it to data-selected. The model still makes the judgement (which descriptor does my prose match?), but the check is mechanical and script-owned: textbook §H, and it catches the retro's exact tell ("the note describes at-bar behaviour but carries a below-bar number"). A self-reported consistent/reconciled token was rejected as a ritual — a model satisfies it by stamping every dimension without reading. On mismatch the gate does not hard-fail: it requires a non-empty data-score-rationale naming why the number departs from the prose (e.g. a red flag that dragged the score down) — the same disagreement→defence shape as the reco-vs-modal gate, so there is one pattern to learn and no incentive to write the note to match the number. Grill Q8 + Q9 (chose level-match+script-compare over the verdict token; chose rationale-on-mismatch over hard-fail-every-mismatch / advisory).
D11 Output (b) gets the method, not the gate. interviewer-notes-skeleton.html has no scoring anchors — no data-dim, no data-v, no data-selected — so check-scoring-calibration.mjs structurally cannot run against it. But (b) makes below-bar claims about interviewers from the same transcript with the same reading-order risk, so the sweep-then-conclude order and the adversarial below-bar pass apply to Phase Coach as method. Fix the bias everywhere; add machinery only where there is something to check. Giving (b) its own scoring surface is explicitly deferred. Grill Q6 (chose method-yes-gate-no over adding a scoring surface to (b) / scoping the epic to (a) only).

Invariants (cross-cutting; the build must not break)

  • INV-1 — No silent uncalibrated scoring. A dimension is never scored against an absent bar. Where descriptors exist they are quoted (FR-4); where they do not, D8's synthesize→write→agree path runs first (FR-7). The original "absent anchor → byte-identical prior behavior" degrade path is deliberately retired — it was the bar-inflation hole itself. Partial anchoring stays valid across dimensions and invalid within one (FR-2).
  • INV-2 — §H is respected, with no gate wearing a presence costume. Every hard condition is presence, ordering, or an integer comparison the script performs; every arithmetic condition (modal, weighted, renormalized, untested-%, band comparison) is computed by check-scoring-calibration.mjs and never by the model. Where a genuine judgement is required, the model records a machine-comparable value (D10's level integer), not a self-graded verdict.
  • INV-3 — The citation gate is untouched. The skill's INV-1 (verbatim ≥40-char transcript-tier citations, blocking STOP-before-done) keeps its exact semantics. The calibration gate is a second gate run alongside it; --stamp changes who writes the proof comment, never what the check verifies. The grounding half held in the retro and must not regress.
  • INV-4 — No fabrication anywhere in the new surface. The evidence sweep cites real transcript spans under the existing citation rules (contiguous single-speaker, normalized single-line view); an untested competency is tagged, never inferred; a missing observer note is stamped draft, never written for the observer. A synthesized rubric (D8) is the one authored-by-the-skill artifact, and it is always labelled as such via data-rubric-provenance — never passed off as the operator's own bar.
  • INV-5 — Confidentiality preserved. The new blocks carry candidate content and live inside the storage root under the existing storage.sh gitignore guard. No new external call, no new network dependency; check-scoring-calibration.mjs is local and offline like its siblings.
  • INV-6 — All 8 archetypes covered, none regressed. The new anchors are additive to the competency data-dim pass, so work-history's extra role-evidence / trajectory-synthesis families are unaffected and the other seven gain the calibration surface uniformly. The level-verdict → reco relationship (design D9) is unchanged.
  • INV-7 — skill-patterns §A–§L conformance. One-fact-one-home (the anchor contract lives in scorecard-skeleton.html; both skills cite it, neither restates it); §J phase anchors (#score, #ground, #resolve, #coach, #scoring-sheet) stay stable so cross-references resolve; §I — no new user-facing flag on /interview-feedback (--stamp is a script argument, not a skill CLI surface, so argument-hint is unchanged); the frozen non-interactive block stays byte-identical.

Open questions (must be settled in the named story's /plan)

  • ⚠ Per-seniority descriptor variants — story 1a4, blocking its authoring. level-ladder.md already varies competency weights across apm · pm · senior-pm · group-pm · director · vp. "Solid, some prompting OK" plainly means something different for an APM than for a Director. If descriptors must vary by seniority too, 1a4 goes from 188 to ~1,128 descriptors and needs re-scoping (or a parameterized descriptor with a seniority-sensitive clause). Settle this before a single descriptor is authored.
  • The 30% untested-coverage threshold (D7) is a proposal, not calibrated against real rounds. Ship it as one named constant in check-scoring-calibration.mjs so it is trivially tunable, and revisit after first use.

Accepted risks

  • Headless runs can score against a bar no human ever saw (D8). The maintainer chose AUTO-PICK over DEFER+provisional for the rubric-agreement gate under --non-interactive, trading a guarantee for unattended throughput. Mitigated — not eliminated — by data-rubric-provenance="synthesized-auto" on the sheet, which makes the provenance auditable after the fact.

Non-goals

  • No change to the citation gate's semantics, tier model, or ≥40-char verbatim rule — only who writes its proof-of-pass comment (INV-3).
  • No re-scoring or migration of already-produced scorecards. The gates apply to new runs; existing artifacts are left alone.
  • No new user-facing flag on /interview-feedback; argument-hint is unchanged (INV-7).
  • No change to competency weights, the level ladder, --level-rubric, or any weight arithmetic — this epic calibrates how a dimension is scored, not how dimensions are weighted. (D7 renormalizes an existing weight set; it does not reassign weights.)
  • No scoring surface for output (b) — deferred (D11).
  • No multi-candidate comparison, no ATS integration, no comments/overlay substrate — the standing /interview-feedback out-of-scope list (design §15) is unchanged.
