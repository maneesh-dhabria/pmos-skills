Design — define-worktree-exit

Epic: #0015 · Route: skill · Plugin: pmos-toolkit · Define depth: lean (the contract is story #0014's acceptance criteria, adopted verbatim — no fresh requirements/grill pass).

This is a singleton epic wrapping one well-specified story (#0014). It exists only to give the three-loop machinery a release unit. The real design contract is story #0014's acceptance criteria; this page records the cross-skill coherence invariants the story's /plan cites by anchor, nothing more.

Problem

The define loop (feature-sdlc/SKILL.md #define-mode step 5) ends a completed definition with: path-scope check → docs-only merge of define/<epic-id> to main → epic → defined → STOP. There is no ExitWorktree and no worktree removal. The session is left parked inside define/<epic-id>, so the next loop (build) runs from the stale define worktree, stories build on the define branch instead of fresh feat/<story-id> worktrees, and /complete-dev --epic can no longer fast-forward (observed live on epic 0011 — recovered with --no-ff). A secondary gap: no documented owner removes the define/<epic-id> worktree (the #epic-train cleanup removes only per-story worktrees).

Approach

Mirror the pattern define already lacks but /complete-dev Phase 16a has: exit-then-remove at the terminal completed step. The fix is localized to #define-mode step 5, applies to both route variants (their worktree/merge/STOP semantics are shared per the line-423 note), and adds a documentation cross-reference so worktree teardown has exactly one named owner.

Cross-skill invariants (cited by the story plan)

I1 — terminal-only teardown. Exit + remove fire only on the completed docs-only merge. A paused define run (not merged) still retains its worktree for --resume. Step 5 is by definition the completed-merge path; paused runs never reach it.
I2 — both route variants. The route: feature and route: skill define flows share worktree/merge/STOP semantics (SKILL.md line 423), so the new exit/cleanup lands once in the shared step 5 and covers both.
I3 — single owner. define step 5 owns define/<epic-id> teardown (preferred per AC). complete-dev #epic-train step 6 keeps owning per-story worktrees only, and gains a cross-reference note stating the define worktree is not its responsibility.
I4 — platform-correct nudge. The STOP log line uses the per-platform execute_invocation mapping (_shared/platform-strings.md), e.g. cd <root> && /feature-sdlc build --next.

Acceptance contract (verbatim from story #0014)

  • After the docs-only definition merge in #define-mode step 5 (both route variants), define calls ExitWorktree back to the root/main checkout before the STOP log line.
  • The redundant define/<epic-id> worktree is removed (output already on main) — or, if retained, the STOP line says it's safe to remove and where to cd. Pick one and document it.
  • A documented owner exists for define/<epic-id> teardown — define removes it at step 5 (preferred), or #epic-train step 6 is extended to also remove it.
  • The STOP log line nudges toward root (platform-correct invocation).
  • Resume safety preserved: only the terminal completed merge triggers exit/cleanup.
