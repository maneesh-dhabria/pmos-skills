Verbatim prompt text (authoritative)

Practitioner+book naming prompt

List 6–10 named practitioners, authors, or institutional voices canonically
associated with the topic "<topic>". Also list 3–5 canonical books, courses,
or textbooks on the same topic. For each item, include a one-line note on
why it's canonical (their seminal work / well-known framework / etc.).

Return as JSON only (no prose):
[
  {"name": "<Full Name|Org|Book Title>", "kind": "practitioner"|"book"|"course",
   "why_canonical": "<one line, ~10-25 words>"}
]

Be specific — "industry experts" is not a practitioner. Prefer named
individuals over publications. For books, give the book title (not the
author) as the name; put the author in why_canonical.

Topic-richness prompt

Topic: "<topic>"

Is this topic broad enough to support all three of:
(a) at least one named framework or model,
(b) a meaningful decision-guide for practitioners (multiple valid paths),
(c) at least one worked example or case study.

Return one of:
- "rich" — supports all three; a primer can be a useful map.
- "narrow-by-design" — a real topic, but supports at most (a) or (c); no
   meaningful decision-guide exists because there are no live tradeoffs.
   A primer is still useful, just shape-different.
- "thin" — too narrow to support any of the three; a primer would be
   padding. If returning thin, ALSO return 2–3 broader reframings the
   user could pick from (each must be a real, broader topic — not just
   a rephrasing).

Output format:
{"verdict": "rich"|"narrow-by-design"|"thin",
 "rationale": "<1 sentence>",
 "reframings": ["...", "...", "..."]  // present only when verdict == "thin"
}

Curator-lens worked-examples paragraph

(Inserted between existing "Cite every empirical claim…" and "Match the reader's vocabulary level…" bullets.)

- For each H2 section, try to surface at least one worked example — a named
  company, named product, named incident, or a clearly-labeled hypothetical
  scenario (prefix hypothetical examples with "Hypothetical: "). Skip when
  no canonical example exists — do NOT invent. Examples make curation
  legible; without them, the section reads as abstract synthesis.

Risks & mitigations

  • R1 — Hallucinated practitioners on obscure topics. Mitigated by S-FR-1.6 drop-silently rule + OQ log. Residual risk: a hallucinated name with a real but irrelevant URL slips through (e.g., another person of the same name). Mitigation deferred to R10 reviewer's on-topic judgement.
  • R2 — Richness classifier false positives on niche topics. Mitigated by the narrow-by-design escape valve. Residual risk: a genuinely thin topic returns narrow-by-design and ships a thin primer; reviewer's R3 (no-hand-wavy-claims) is the catch.
  • R3 — Deep-tier source-floor (15) unachievable on niche topics. Mitigated by existing retry-once + thin-source disclosure path; same as current behavior just at a higher threshold.
  • R4 — Latency at deep tier. Naming step + per-practitioner queries multiply Phase-2 latency. Acceptable: deep tier is the explicit user choice for thoroughness over speed.
  • R5 — practitioner_index audit clutter. Dropped entries retained per D1; if this proves noisy, future iteration can add a flag to suppress. Not in scope here.
