Plan — /frameworks

Date: 2026-06-07 · Source: 02_spec.html · Execution mode: inline (controller-driven; bulk Stage-B steps fan out to parallel subagents) · Tier: 3.

Per skill-mode discipline: no version-bump / changelog / README / manifest tasks appear in any wave — those are /complete-dev release prerequisites (see spec §release-prereqs).

Wave 1 — Skill scaffold (no corpus yet)

  • T1 — SKILL.md. Frontmatter (single-line description w/ ≥5 triggers; user-invocable: true; full argument-hint) + body: intro, command surface, ## Platform Adaptation, ## Track Progress, ## When NOT to use, the byte-identical non-interactive block, phase sections (Retrieve / Browse / Sync), ## Anti-Patterns, and a final ## Phase 6: Capture Learnings (literal heading) emitting the one-line learning. Acceptance: skill-patterns §A/§B/§D structural checks pass.
  • T2 — reference/ docs. corpus-schema.md, situation-taxonomy.md, ingestion.md (incl. the /diagram batch contract), matching.md (incl. --json shape). Each with a ## Contents TOC (skill-patterns §C). Acceptance: SKILL.md links resolve to each.

Wave 2 — Stage-A deterministic scripts (TDD: test first)

  • T3 — split-corpus.mjs. Test first on a real Decision-Making fixture (the page already fetched): asserts 9 frameworks split with correct name/body_md/references/author/commentary. Zero-dep Node, --selftest.
  • T4 — derive-fields.mjs. Test: rejects unknown problem_tags/decision_type; merges LLM field JSON into records; sparse records still validate. --selftest.
  • T5 — validate-corpus.mjs. Test: coverage thresholds, dangling related/refs, exit codes (0 pass / 1 fail). --selftest.
  • T6 — match.mjs. Test: weighted scorer ranks an obvious problem to the right framework; confidence floor on nonsense; cap ≤5; deterministic ordering. --selftest.
  • T7 — build-library.mjs. Test (.sh, run ≥2×): emits self-contained index.html — no http(s):// asset refs, SVGs inlined, filter controls + search present, inline-expand detail with "PM's take" block.
  • T8 — json-contract.test.mjs. FR-JSON shape over a fixture corpus (the standalone-consumer proof): keys/types, cap≤5, score∈[0,1], JSON-only stdout.

Wave 3 — Situation taxonomy authoring

  • T9 — data/situations.json. Author the closed problem_tags registry + ~15–25 PM situations from the 22 Notion categories, each mapped to framework ids (grouped by super_category). Acceptance: validate-corpus.mjs finds 0 dangling situation→framework refs once the corpus exists.

Wave 4 — Full corpus population (Stage-B; the heavy wave)

  • T10 — Fetch. notion-fetch all 22 category pages → raw markdown temp files (slice large ones past the token cap).
  • T11 — Split. Run split-corpus.mjs over all 22 → ~200 raw records.
  • T12 — Derive fields. Parallel subagents (≈N frameworks each, strict output contract per the bulk-lookup learning) emit match-fields for all ~200; derive-fields.mjs validates + merges.
  • T13 — Diagrams. Generate ~200 owned SVGs via /diagram … --non-interactive --approach "<framing>" --theme technical --out data/diagrams/<id>.svg --on-failure ship-with-warning, fanned out in waves at full rigor. --changed-only cache seeded. Cost checkpoint with the user before launching the full fan-out.
  • T14 — Assemble + validate. Write data/frameworks.json; validate-corpus.mjs → coverage report (≥95% name+body+refs, 100% diagram or logged exceptions, 0 invalid tags).
  • T15 — Build library. build-library.mjs → data/… → shipped index.html.

Wave 5 — Verify the populated skill

  • T16 — Browser smoke. Serve index.html; exercise filter + inline-expand + a search query with geometric/state assertions (offline, file://).
  • T17 — Precision check. /frameworks "<problem>" over ≥10 sample problems; author-judge apt top-5 (target ≥8/10).
  • T18 — Structural gate. structure.test.sh + a dry-run of the skill-eval rubric (skill-patterns §A–§F) before Phase 6a.

Risks & mitigations

  • 200 full-rigor /diagram runs is the dominant cost. Mitigation: fan out in parallel waves; --changed-only cache; --on-failure ship-with-warning so no single failure aborts; explicit cost checkpoint before T13.
  • Notion page token-cap on fetch. Mitigation: save-to-file + slice (the Feature-Prioritization page already exceeded the cap this session).
  • SIGPIPE-flaky .sh tests. Mitigation: materialize-then-here-string; run each ≥2×.
  • Worktree split-brain. Mitigation: all edits via worktree-absolute paths; already EnterWorktree'd.
