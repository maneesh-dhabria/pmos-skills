Plan — magazine output UX (Wave 1)

Created using pmos‑toolkit

Approach

One file (scripts/render-issue.js), so tasks are sequential. TDD-flavored: the inline selftest() is grown alongside each change and re-run after every task (node render-issue.js --selftest). execution_mode: inline. Each task is one focused commit where practical; tasks sharing the same file are batched when surgical overlap is high (documented deviation).

Wave 1 — markup & CSS foundation

  • T1 (FR-2/FR-4 markup): in cardHtml() add data-guid, data-type, the feed-dot (deterministic hashHue(feed)), and a .mark ✓ button. Add hashHue() helper. Update selftest: a card has data-guid + data-type; output has feed-dot. Verify: --selftest PASS.
  • T2 (FR-1/FR-4/FR-6 CSS): extend ISSUE_CSS — type left-border accents, .feed-dot, pastel .chip via --h, .card.read dim, body[data-view] grid/carousel/listicle modes, .viewsel/budget styles. Additive only. Verify: --selftest PASS (CSS string present).
  • T3 (FR-6): bake style="--h:${hashHue(tag)}" into each tag chip in cardHtml(). Update selftest: a chip carries --h. Verify: PASS.

Wave 2 — per-card affordances

  • T4 (FR-7): wrap bullets in ul.bullets data-count; emit button.more only when >3 bullets; CSS clamp + .expanded; delegated JS toggle. Update selftest: 5-bullet item has more, 3-bullet item does not. Verify: PASS.
  • T5 (FR-5): add parseMinutes(); compute skim/audio sums in renderIssue(); render the budget line in the header (omit empty clauses). Update selftest: mixed fixture header contains "skim" + "audio". Verify: PASS.

Wave 3 — interaction layer (ISSUE_JS)

  • T6 (FR-1): view-switcher control in header + ISSUE_JS view logic (read/persist mag:view, apply body[data-view], carousel index + ◀▶). Update selftest: output has data-view + mag:view. Verify: PASS.
  • T7 (FR-2): read-state JS — load mag:read:<date>, apply read, .mark/m toggle + persist, header counter #left, #hideRead toggle. Update selftest: output has id="left" + mag:read:. Verify: PASS.
  • T8 (FR-3): single keydown handler (j/k/o/f/m + carousel arrows), ignoring input/select focus. Update selftest: handler references 'j'/'k'. Verify: PASS.

Wave 4 — filter rework

  • T9 (FR-8): replace fieldset markup with dropdowns (feed/tag multi, type, read-state) + #count; rewrite magFilter() preserving AND/OR semantics + date range; live counter. Update the existing filter assertions + add f-type/f-read/id="count". Verify: --selftest PASS (all preserved + new assertions).

Wave 5 — docs & final verification

  • T10: update reference/issue-format.md — document the view-switcher, read-state, dropdown filter + counter, catch-up budget, type/tag/feed color cues, and the new localStorage keys. Note NFR-1/NFR-2 still hold.
  • T11 (final verify): run node render-issue.js --selftest; render a real fixture, open from file://, manual spot-check of all FRs + JS-off grid; grep the output for external asset/src URLs (expect none beyond read links).

Risks & mitigations

  • Filter rewrite breaks selftest (T9): update assertions in the same task; keep f-feed/f-tag class hooks.
  • JS regressions blank the issue: all handlers null-guard; grid is the no-JS fallback (NFR-2) — verified JS-off.
  • Visual clutter: restrained defaults (D3); re-check skim feel on the real fixture at T11.
  • Empty-issue / missing-field edge cases: handlers no-op on absent elements; parseMinutes returns 0 on unparseable input.
