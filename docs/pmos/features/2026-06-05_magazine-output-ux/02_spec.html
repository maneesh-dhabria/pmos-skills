Spec — magazine output UX (Wave 1)

Created using pmos‑toolkit

Architecture

All changes are confined to scripts/render-issue.js: the ISSUE_CSS string, the ISSUE_JS string, the cardHtml() emitter, the renderIssue() assembler, and the inline selftest(). No new files; renderLibrary() is untouched except for shared CSS it already references. The output stays a single self-contained HTML document (inline <style> + <script>); no external assets, no server (NFR-1). All JS is loaded once at page end and attaches behavior to server-rendered DOM, so the grid is fully functional with JS off (NFR-2).

State model (client-side only): localStorage keys — mag:view (one of grid|carousel|listicle), mag:hideRead (0|1), and mag:read:<issue_date> (a JSON array of read guids). No server, no sidecar files.

cardHtml() changes

  • data-guid (FR-2): add data-guid="${esc(it.guid||it.link||it.title)}" to the <article> — the stable read-state key.
  • data-type (FR-4): add data-type="${esc(it.type||'newsletter')}" for CSS type styling.
  • data-feed-hue (FR-4): compute a deterministic hue from the feed name (hashHue(it.feed)) and emit a per-card feed dot <span class="feed-dot" style="--h:${hue}"></span> in the meta line.
  • Tag color (FR-6): emit each chip with style="--h:${hashHue(tag)}"; CSS turns --h into a soft pastel background.
  • Bullet clamp (FR-7): wrap bullets in <ul class="bullets" data-count="${n}">; emit a <button class="more"> only when n > 3. CSS hides bullets beyond the 3rd unless .expanded.
  • Mark-read control (FR-2): add a small <button class="mark"> (✓) to each card; JS toggles a read class + persists.

hashHue(s): a tiny deterministic string hash → integer 0–359 (e.g. djb2 % 360). Pure function; same tag/feed → same hue across issues (FR-4/FR-6). Lives in both Node scope (for any server-side use) and the inline JS is not needed since hue is baked at render via inline --h.

renderIssue() header changes

  • Catch-up budget (FR-5): a parseMinutes(reading_time) helper extracts the leading integer ("6 min" → 6, "101 min listen" → 101). Sum newsletter minutes → skim total; sum podcast minutes → audio total. Render "~S min to skim · Hh Mm of audio" in the header, omitting either clause when its sum is 0.
  • View-switcher control (FR-1): a segmented control (<div class="viewsel"> with three <button data-view>) in the header.
  • Reading counter + hide-read (FR-2): a header span <span id="left"> ("N of M left") and a <label> checkbox #hideRead.

Filter bar rework (FR-8)

Replace the <fieldset> feed-checkbox + tag-chip markup with compact controls:

  • Feed: a <select multiple class="f-feed"> (or a button-dropdown of checkboxes) — keep class f-feed as the JS hook.
  • Tag: a multi-select <select multiple class="f-tag"> — keep class f-tag.
  • Type: a <select class="f-type"> (all / newsletter / podcast).
  • Read-state: a <select class="f-read"> (all / unread / read).
  • Date range: unchanged (#f-from/#f-to).
  • Counter: <span id="count"> updated by magFilter() to "showing X of Y".

magFilter() is reworked to read the new controls but keeps the existing logic shape: a card is visible iff it matches feed (OR within selected) AND tag (OR within selected) AND type AND read-state AND date range. Hidden cards keep the hidden class. The counter is set to the visible count.

ISSUE_JS additions

  • View control: on load, read mag:view (default grid), apply document.body.dataset.view; clicking a [data-view] button updates it + persists. CSS keys off body[data-view=...].
  • Carousel: when view=carousel, show one non-hidden card at a time; ◀▶ buttons + ←/→ keys move an index over the visible set.
  • Read-state: load read-set from mag:read:<date>; apply read class to matching cards; clicking .mark or pressing m toggles + persists + updates the counter; #hideRead toggles a body class + persists.
  • Keyboard: a single keydown handler implements j/k/o/f/m and arrows (carousel), ignoring events when focus is in an input/select.
  • Bullets: delegated click on .more toggles .expanded on the card.
  • All handlers no-op safely if their target elements are absent (robust to empty issues).

ISSUE_CSS additions (restrained, FR-3 visual)

  • .card[data-type=newsletter]{border-left:3px solid var(--accent)} / .card[data-type=podcast]{border-left:3px solid #7c3aed} — subtle type accent.
  • .feed-dot{width:.5rem;height:.5rem;border-radius:50%;background:hsl(var(--h),60%,55%)}.
  • .tags .chip{background:hsl(var(--h,210),70%,92%);color:#333} — soft pastel tag tint.
  • Bullet clamp: .bullets li:nth-child(n+4){display:none} / .card.expanded .bullets li{display:list-item}.
  • View modes via body[data-view=listicle] .grid{grid-template-columns:1fr} and body[data-view=carousel] .grid{display:block} + carousel card visibility class.
  • .card.read{opacity:.55}; body.hide-read .card.read{display:none}.
  • View-switcher + budget styles; all additive, no change to existing rules.

Testing strategy (selftest additions)

Extend the inline selftest(). Update the existing filter assertions to the new markup (still assert f-feed + f-tag hooks exist; add f-type, f-read, id="count"). Add one assertion per FR:

  • FR-1: output contains data-view buttons + mag:view persistence code.
  • FR-2: a card has data-guid; output contains id="left" + mag:read:.
  • FR-3: keydown handler references 'j'/'k'.
  • FR-4: a card has data-type; output contains feed-dot.
  • FR-5: a two-item fixture (newsletter "6 min" + podcast "101 min listen") yields a header containing "skim" and "audio".
  • FR-6: a tag chip carries a --h hue style.
  • FR-7: a 5-bullet item emits a class="more" button; a 3-bullet item does not.
  • FR-8: output contains f-type, f-read, and id="count".
  • Preserved: top-picks lane, pick/degraded classes, dedup, pmos:skill meta, Listen/Read verbs.

NFR-2 is asserted indirectly: the grid markup (cards, bullets, links) is present in the static HTML independent of the script block.
