Overview

Build the inline-comments overlay + sidecar persistence + /comments resolve CLI + per-skill apply-edit-at-anchor instrumentation for the 14 HTML-emitting surfaces of the pmos-toolkit pipeline. Architecture is Finalist A-hybrid (always-serve over http://localhost, dual-anchor capture, per-thread originating-skill subagent dispatch). Build order tracks spec §15.1: spike → substrate → /spec tracer bullet → resolver completeness → fanout to remaining 12 skills + 2 orchestrator surfaces → SVG retrofit + polish → verify integration.

Constraints discovered

  • No npm / no bundler. Repo is zero-dep JavaScript (Node 18+ stdlib only). diff-match-patch must be vendored verbatim. comments.js is a plain ES2018 script loaded with defer — no transpile, no module bundling.
  • No JS test framework. Tests are plain node tests/scripts/<name>.test.js files using assert from stdlib, wrapped in bash scripts that set -euo pipefail and grep output. Plan-emitted tests follow this shape.
  • plugins/pmos-toolkit/skills/comments/ does not exist. Created at T10 with SKILL.md + tests/ + scripts/ subdirs.
  • serve.js's existing --port-file flag is referenced by tests/scripts/assert_serve_js.sh — the rename in T4 must preserve the alias for one release cycle (P2) to keep that test passing without modification.
  • Chrome FSA requires secure context. http://localhost qualifies; http://192.168.x.x does not. Hence FR-45's strict 127.0.0.1 bind (NFR-07 too).
  • 14 emit surfaces, 15 instrumentations. FR-21 enumerates: 13 originating skills + 2 orchestrator artifacts. /wireframes emits a subfolder of N HTMLs + 1 index.html — counts as ONE originating-skill instrumentation surface (one entrypoint, one bake-edit in the wireframe template) but the bake reference goes into N+1 files. Per FR-21 math: 13 skills + 2 orchestrator = 14 entities / 15 surfaces. Coverage check (T27) asserts 13 contract tests + 15 emit references (the 14th = /wireframes' index.html, 15th = the per-wireframe template — but both share one bake-edit since they share a template helper).
  • 2 orchestrator surfaces are /feature-sdlc only. 00_pipeline.html and 00_open_questions_index.html are both written by /feature-sdlc. /ideate can also write the OQ index in skill-feedback mode, but the bake edit lives in /feature-sdlc's template (single source of truth). T21 covers this.
