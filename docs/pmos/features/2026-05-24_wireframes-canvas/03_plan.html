Risks

  • R1 — Iframe scroll bleed: if a per-device HTML file is taller than the canvas screen window, scrolling inside the iframe propagates to canvas pan. Mitigation: set iframe wrapper to overflow: hidden and disable iframe wheel events at zoom < 1×.
  • R2 — DESIGN.md journey format drift: if a future /wireframes-bootstrap changes journey heading conventions, arrow parsing breaks. Mitigation: tolerate both ## Journey: and ### Journey N — <name>; warn but don't fail if no journeys found.
  • R3 — CDN unavailable: first-time viewing requires internet. Mitigation: SRI hashes mean tampering is caught; an offline fallback (local-vendor panzoom + leader-line) could be added later — not in v1 scope.
  • R4 — Large feature folders: >50 screens × 3 devices = 150 iframes. May be slow at zoom-1×. Mitigation: loading="lazy" on iframes + default zoom 0.3× so most are below the lazy-load threshold.
