Requirements — /one-on-ones

pmos-managerkit · epic 260702-z77 · route: skill · Tier 2 · from shape brief 2026-07-02_effective-one-on-ones.html

Problem

A manager has no durable, low-friction, per-report thread that assembles a tailored, human-first agenda at the moment each 1:1 starts and carries continuity across sessions — open action items, recurring themes, goals, and manager-recorded feedback — so weekly 1:1s stop decaying into status updates and growth/morale get tended. (Full framing, decomposition, and competing framings: the shape brief; Framing B — "per-report relationship thread" — chosen.)

User & jobs

User: any manager with recurring direct reports. Jobs: (1) register a report; (2) capture a discussion item the instant it occurs; (3) prep a human-first, continuity-aware 1:1; (4) log what happened and close loops; (5) keep each report's goals + performance/coaching feedback in view for planning; (6) run a periodic deeper career conversation.

Functional requirements

#RequirementPriority
FR1Add report — register a direct report; anchor identity in /people (offer to create when absent; never fabricate); create the per-report 1:1 record (cadence, role, goals, operating-manual).MUST
FR2Quick-capture discussion item — one command appends an item to the report's running-agenda inbox; unattended-safe; optional intent tag.MUST
FR3Plan/prep a 1:1 — assemble a human-first agenda from header (goals, themes, feedback) + open action items + inbox; surface actively-coached suggestions (intent-tagged questions, human-first opener, status-creep + stale-action-item flags). Emit a commentable HTML prep artifact.MUST
FR4Log a session — append a dated entry (topics, decisions, action items w/ owner+status, questions) newest-first; clear discussed inbox items.MUST
FR5Context management — set/update goals, standing themes, performance feedback, coaching feedback in the persistent header (manager-entered only).MUST
FR6Career conversation — Laraway 3-part deeper session, offered separately from weekly prep; writes a career-plan block.SHOULD
FR7Overview — bare /one-on-ones lists reports, who's due, and open items.SHOULD
FR8Coaching corpus — bundled reference: intent-tagged question bank + named models + durable principles, honoring the 4 attribution caveats.MUST

Constraints & non-functional

  • Independence (hard): fully independent of /interview-feedback — no import, no _shared cite, no scorecard read.
  • Anchors: identity via /people (~/.pmos/people); no /mytasks dependency.
  • Privacy: records under ~/.pmos/one-on-ones/, outside the repo — never committed.
  • Zero-dep Node scripts + html-authoring substrate for the prep artifact (patterns reused from /people, /mytasks, /interview-feedback — patterns only, no code coupling).
  • Non-interactive: note/log run unattended (Recommended → AUTO-PICK); missing identity or session body DEFERs, never fabricated.
  • Conformance: skill-patterns.md §A–§L; skill-eval ([D]+[J]); 4 hygiene lints + audit-recommended green.

Out of scope (v1)

  • Pulling feedback/scorecards from /interview-feedback or any other skill (manager types feedback in).
  • Task management of reports' work (no /mytasks integration).
  • A full multi-report web viewer (the prep artifact is per-session HTML; a browsable viewer is a possible follow-up).
  • Scheduling/calendar integration; reminders/notifications.
