Epic 0020 — Concurrency-safe backlog ids

Define-loop design contract · route: skill · pmos-toolkit · 2026-06-12 · date+short-rand ids + merge-time uniqueness gate + derived INDEX

The incident (root cause, observed live)

Two /skill-sdlc define sessions ran in parallel, both branched off the same main at 137b929 (where max(id) == 0015). Each allocated ids independently per _shared/tracker-crudl.md §2:

id: 4-digit zero-padded sequential integer. Per-store counter; no global coordination. Allocate max(existing id) + 1.

So both minted epic 0016 + story 0017 — one for /frameworks browse-UI fixes, one for the /book-summary skill. The collision is silent because of two compounding facts:

  1. Slug-suffixed filenames mask the clash. 0016-frameworks-browse-ux.md vs 0016-book-summary-skill.md are different paths — git sees two new files, no path conflict, clean merge. Two items then both declare id: 0016 and nothing flags it.
  2. INDEX.md is hand-edited, so it is the only artifact that can conflict — but it conflicts as noisy adjacent-row text, or merges cleanly into a duplicate 0016 row. Either way the index stops bijecting to ids.

A third latent failure: the branch name define/<id> is id-derived, so two non-harness defines both want define/0016 (the second git worktree add errors). The book-summary run dodged only because it used a harness-native worktree with a different branch name. This epic was itself allocated 0020 by hand (skipping book-summary's in-flight 0018/0019) — the exact manual coordination this fix is meant to remove.

Coherence invariants

  • No global counter, no central coordinator. The fix must work for N parallel worktrees AND separate clones (the origin + gitlab-mirror topology) without a shared lock.
  • Existing ids are immutable. Legacy 4-digit serials (0001–0019) are grandfathered, never rewritten — they are referenced by parent:, dependencies:, branch names, claim locks, and released changelog entries.
  • Ids stay filename-, branch-, and lock-safe. Any new id must be valid in {id}-{slug}.md, define/{id} / feat/{id}, and backlog/claims/{id}.
  • Human-readable + roughly chronological. The maintainer reasons in spoken ids ("0017"); the replacement must stay speakable and time-orderable.

The fix — three independent layers

UUID-style ids alone are necessary but not sufficient; the durable fix is three layers, each useful on its own.

Layer 1 — collision-free id scheme (date + short-rand)

D1 — id format. New ids are <MMDD>-<rand> where <rand> is 3 lowercase Crockford-base32 chars (alphabet 0-9 a-z minus i l o u to avoid look-alikes). Example: 0612-k3f. Per-day space = 28³ ≈ 22 000 — ample for a personal backlog; the merge-time gate (Layer 2) is the backstop against the astronomically rare same-day collision. The embedded MMDD is a human hint, not a sort key (year wraps); true chronology comes from the existing created: field.
  • Coordination-free: each session mints its own id from the date + randomness — no max+1, no counter, no shared lock. Works across worktrees AND clones.
  • Unambiguous vs legacy: the - separator distinguishes new ids from grandfathered 4-digit serials at a glance and in regexes.
  • Generation must avoid Math.random() in skill scripts where that is banned (resume determinism) — generate from a seeded/process source in a tiny tool, or accept the id as an input. Pin the exact generator in build.
  • Allocation sites updated: backlog/SKILL.md #add step 2 and the define/build id mints stop calling "Allocate max+1"; _shared/tracker-crudl.md §2 becomes the single home of the new format (cross-plugin sync via scripts/sync-shared.sh --from=pmos-toolkit).

Layer 2 — merge-time id-uniqueness hard gate

D2. The define definition-merge (feature-sdlc/SKILL.md #define-mode step 5) already runs a deterministic path-scope check. Add an adjacent id-uniqueness hard gate: before merging, refuse if any item id introduced on the branch already exists on main; after merge, assert INDEX has no duplicate ids. Converts silent corruption → loud refusal with the offending ids. Worth having regardless of the id scheme — a backstop for any future allocator bug (and for the same-day random collision).

Layer 3 — INDEX.md as a derived artifact

D3. Stop hand-editing INDEX.md in the define merge. The /backlog rebuild-index verb already regenerates it by scanning backlog/items/*.md; make the define merge (and #add) regenerate rather than hand-append rows. This removes INDEX as an independent corruption/merge-conflict surface. It cannot invent uniqueness (Layer 1's job) but it makes the index always reconcilable from the items, and it should sort by created: desc so chronology survives the non-monotonic ids.

Migration stance

  • No rewrite of existing ids. Legacy serials and new date+rand ids coexist; readers accept both (^(\d{4}|\d{4}-[0-9a-hj-km-np-tv-z]{3})$).
  • The book-summary 0018/0019 renumber already done in its session stays as legacy serials — no back-conversion.
  • First post-fix rebuild-index run normalizes INDEX sort to created: desc.

Acceptance criteria (story 0021)

AC1 — New backlog ids are minted as <MMDD>-<rand3> (Crockford base32, no i/l/o/u); no allocation reads max+1 or any shared counter. Two simulated parallel mints off the same baseline produce different ids.
AC2 — Legacy 4-digit ids still parse and are never rewritten; the id validator accepts both forms.
AC3 — The define definition-merge refuses (loud, lists offending ids) if a branch id already exists on main; post-merge it asserts INDEX has no duplicate ids.
AC4 — INDEX.md is regenerated (not hand-appended) by the define merge and by #add, sorted by created: desc; no merge-conflict-prone hand edits remain.
AC5 — All id-consuming sites (filename, define/feat branch, claim lock, parent/deps) verified scheme-agnostic; scripts/sync-shared.sh --from=pmos-toolkit propagates the tracker-crudl change.
AC6 — Skill rubric green (skill-eval) + backlog skill tests updated to the new id format; /backlog add still single-tool-call frictionless.

Out of scope

  • Back-converting existing serial ids to the new scheme (grandfathered).
  • A cross-clone distributed counter / ref-CAS allocator (rejected — see D-rejected).
  • Changing /backlog next prioritization logic.
