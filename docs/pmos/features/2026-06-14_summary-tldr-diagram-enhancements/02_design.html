Decisions

D1 — obs-2 (emit-before-diagram) structurally subsumes the blocker (F1+F2 → one fix)

The blocker is a data-loss risk: the Phase 4/5-approved summary lives only in conversation context across the multi-turn diagram loop, and a compaction loses it. Rather than the retro's proposed {out}.summary.tmp side-file, the canonical fix is to reorder the phases: emit the full summary HTML to disk first, then run /diagram and inject the validated SVG into the on-disk artifact. The approved text is persisted in the real artifact before any long-running work, so it is crash-safe by construction — and no redundant temp file is introduced (the artifact is the single source, §K one-fact-one-home). One fix resolves F1 and F2.

Anti-pattern to encode: never run the slow diagram loop between Phase 5 approval and the first on-disk emit.

D2 — obs-1 (h1 title) is OUT of this epic; deferred to epic 260613-ev1

A substrate-level body <h1> is already owned by epic 260613-ev1 ("Pipeline-doc CSS readability refresh", status defined): it converts the current ~12px toolbar title into a breadcrumb and renders a real body <h1> from {{title}} in _shared/html-authoring/template.html for every pipeline artifact. /summary-tldr already supplies a meaningful {{title}} (e.g. "Porter Metric Store POV — TL;DR"). Re-implementing an h1 inside /summary-tldr would duplicate that fix and double-up when ev1 lands. Maintainer decision (2026-06-14, via define grill): defer obs-1 to ev1; no /summary-tldr change. Documented, not silently dropped.

D3 — three independent stories, default one-per-skill (G3)

The change-sets are independent (different files, no inter-story coupling), so the split is the default one-story-per-skill plus one for the shared substrate. No story dependencies:. The substrate story is its own unit because the fix benefits every skill, not just summary-tldr.

D4 — single plugin / release unit (D17): pmos-toolkit

All three stories land in pmos-toolkit. The substrate edit (Story C) is authored canonically in pmos-toolkit and synced to pmos-learnkit via scripts/sync-shared.sh --from=pmos-toolkit at release. The whole epic rides one pmos-toolkit minor bump; /complete-dev (Loop 3) is the sole writer of the version bump, changelog, README rows, and tag. No release-prerequisite work appears in any story's tasks.yaml.
