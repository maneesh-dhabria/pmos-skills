Design — /frameworks corpus expansion & re-found on direct authoring

Epic 260617-4w1 · route: skill · plugin: pmos-learnkit · 2026-06-17 · This is the epic design_doc — the cross-skill coherence contract; per-story /plans cite its anchors.

Context

A corpus-expansion research run (documented in the source repo at learn-magazine/docs/pmos/frameworks/) produced 74 net-new, ready-to-ingest framework records — each an embedded JSON record + a paired, house-style owned SVG — plus a written, repeatable research + authoring process (RESEARCH-PROCESS.md, AUTHORING-TEMPLATE.md). The maintainer wants those 74 merged into the bundled /frameworks corpus, the process captured inside the skill so future expansions are repeatable, and — a re-founding decision — the legacy Notion sync pipeline removed entirely: the corpus is now maintained by direct authoring + the documented process, so Notion is no longer the source of truth.

Today the corpus is bundled and git-tracked (data/frameworks.json = 272 entries, + 421 tracked diagram SVGs). Runtime paths (retrieve / --json / browse / situations) are fully offline over that shipped corpus. Only sync ever touched Notion.

Goal & non-goals

  • Goal. Merge the 74 frameworks into the bundled corpus; capture the research + authoring process as shipped /frameworks reference docs; remove the Notion sync feature, leaving direct authoring as the single corpus-maintenance path.
  • Non-goal — curator commentary. All 74 records ship with commentary: null by design: per AUTHORING-TEMPLATE.md the "PM's take" is the curator's to write personally, never the drafter's. The library already renders entries with null commentary cleanly. Writing the 74 takes is deliberately out of scope (a later curator pass).
  • Non-goal — re-authoring the 74 records. They are accepted as-authored; the build splices them verbatim and relies on validate-corpus.mjs as the gate, only touching a record if a gate genuinely fails on this corpus.

Invariants (cross-skill coherence contract)

Every story's /plan and /skill-eval is bound by these:

  1. Runtime + authoring paths keep working unchanged. match.mjs (retrieve / --json), build-library.mjs (browse), and validate-corpus.mjs (the corpus gate) are NOT sync — they survive intact and their --selftest / tests/ stay green.
  2. The corpus validator's imports never break. validate-corpus.mjs must keep importing DECISION_TYPES, LIFECYCLE_STAGES, validateAnchors from a retained module (D6). node validate-corpus.mjs --selftest passes after every change.
  3. The corpus stays a self-contained, offline, bundled artifact. Diagrams are owned inline SVGs (no http(s)/S3 refs); the 74 new SVGs are git-tracked under data/diagrams/ with the flattened <category>__<name>.svg names the records already reference.
  4. No dangling cites after sync removal. Removing ingestion.md means SKILL.md and corpus-schema.md/situation-taxonomy.md must drop or repoint every reference to it and to the removed scripts (split-corpus.mjs, apply-rederive.mjs, the sync verb). Grep-clean is an AC.
  5. Distribution + anchor gates re-confirmed on the merged 346 corpus — not trusted from the author's local validation.

Functional requirements

Deliverable A — ingest the 74 (story 260617-2gw)

  • FR-A1. All 74 embedded JSON records are merged into data/frameworks.json (272 → 346), via a deterministic script (D2), preserving the existing record shape and stable ordering.
  • FR-A2. All 74 paired SVGs are copied into data/diagrams/ under the exact flattened filenames the records' diagram/diagrams fields reference, and git-tracked.
  • FR-A3. node scripts/validate-corpus.mjs data/frameworks.json data/situations.json exits 0 on the merged corpus — required fields, tag registry, enums, related resolution, diagram_anchors substring + length, distribution gate, ≥95% coverage all green.
  • FR-A4. node scripts/build-library.mjs --out <docs>/frameworks/index.html renders; the new cards appear and each new framework's inline SVG renders at its diagram_anchors position (spot-checked live).
  • FR-A5. Any SKILL.md/reference hardcoded corpus-count reference is updated to 346 (coordinated with Deliverable B per D7).

Deliverable B — re-found on direct authoring (story 260617-kac)

  • FR-B1. The sync command surface is removed: Phase 5 #sync, the Phase 1 dispatch branch, the argument-hint token, the frontmatter Notion/sync language + the "rebuild the frameworks corpus" trigger, the #sync Track-Progress note, and the Notion/S3/--changed-only/write-back anti-patterns + Platform-Adaptation Notion notes.
  • FR-B2. The Notion-only pipeline scripts split-corpus.mjs and apply-rederive.mjs are deleted; their tests (tests/split-corpus.test.mjs) are removed and tests/structure.test.sh is updated so it no longer asserts the removed scripts.
  • FR-B3. The validation exports are preserved (D6 / inv-validate): validate-corpus.mjs --selftest still passes.
  • FR-B4. reference/ingestion.md is removed; SKILL.md, corpus-schema.md, and situation-taxonomy.md drop/repoint every cite to it and to the removed scripts (inv-nodangle; grep-clean AC).
  • FR-B5. SKILL.md cites it as how to grow the corpus.
  • FR-B6. SKILL.md description/body reflect the new reality: a bundled, directly-authored corpus (no Notion sourcing claim), 346 entries.

Risks

  • diagram_anchors re-validation — anchors authored against the source body_md; if any record's body_md was altered in transit, the ≥40-char substring check fails. Mitigation: the splice is verbatim; validator catches any miss → fix the anchor.
  • derive-fields coupling missed — deleting derive-fields.mjs wholesale would break the validator. Mitigation: inv-validate + a --selftest gate in the plan.
  • dangling cites after ingestion.md removal — caught by a grep-clean AC and the /complete-dev route:skill coherence [J] gate at release.
