Risks & how the plan handles them

  • iframe sandbox tokens. Primers are self-contained inline-CSS/JS docs; reading needs no scripts. Use a reading-only sandbox plus allow-popups/allow-popups-to-escape-sandbox for "Open in new tab"; never combine allow-same-origin+allow-scripts (escape risk). Final token set fixed in T2.
  • file:// iframe load. The library page and primers co-locate in {docs_path}/primer/; iframe src is sibling-relative. T6 dogfood verifies it loads under file://; same-directory relative fallback if a browser blocks it.
  • Perf with 60+ primers. Only the opened primer's src is set (lazy) — no eager load. Asserted in T2/T6.
  • Substrate regression. T2 golden-compares the columns-mode emit against pre-change output; T4 re-runs both consumers' shipped suites un-edited.
