Key risks

  • Structure-vs-inference ambiguity. The rule is fixed: structure always wins on conflict; only the unresolved remainder is inferred (T2). The confirm gate (T3) is the safety net — nothing writes unconfirmed.
  • Testing an agent-driven parse. If no pure tokenizer is factored out, fixtures are documented expected-tree scenarios; T6 still asserts the five cases (indentation, markers, tokens, inferred, structure-wins).
  • People never invented. Tokens resolve via /people find; unresolved ones are flagged in the report, not created (T4).
  • Non-interactive safety. AUTO-PICK the recommended reading + record ambiguities as open questions; block stays byte-identical (T3/T5).
