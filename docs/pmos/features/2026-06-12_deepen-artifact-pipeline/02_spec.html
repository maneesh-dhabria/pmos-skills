2. The --depth dial (F8, OQ4 — single dial)

--depth brief|standard|deep is the sole user-facing dial. It controls BOTH stage gating (table above) AND artifact section count (the old --tier job).

  • Resolution: --depth <v> flag > .pmos/settings.yaml :: artifact.default_depth > builtin default standard. Print depth: <v> (source: cli|settings|default) to stderr once.
  • Section mapping (replaces template tier-detection Step 3): brief→template's lite section set; standard/deep→full set. Single-tier templates ignore this.
  • --tier lite|full retired as documented flag → kept as a back-compat silent alias (nl-sugar): --tier lite≡--depth brief, --tier full≡--depth standard. Not machine-coupled (the orchestrator never passes --tier to /artifact), so retiring the documented surface is §I-safe; the alias prevents muscle-memory breakage. Update argument-hint to advertise --depth.
  • NL-first: "deep doc pipeline" / "full battery" ≡ --depth deep; "quick draft" ≡ --depth brief.

3. Child-skill orchestration constraint (resolves FR-22, the #1 risk)

Verified: /polish, /grill, /diagram all ship a --non-interactive block and degrade headless. /grill already takes --depth brief|standard|deep; /diagram already takes --non-interactive --on-failure drop|ship-with-warning|exit-nonzero --out <path>. But skills cannot be invoked by a subagent (/polish states this explicitly; it is true for all three). Therefore:

  • Main-agent run (the normal /artifact … case): Phases 3.5/3.7/3.8/3.9 invoke the child skills via the Skill tool normally.
  • Subagent-dispatched run (e.g. /feature-sdlc dispatches /artifact as a child): the child-skill phases CANNOT run. They degrade to skip-with-note — emit a <!-- pmos:deferred-pass > note in the artifact + a chat line recommending a manual /artifact refine or direct /polish//grill run, and log the degradation. Detection: the Platform-Adaptation "No subagents" clause + a runtime check ("am I able to call Skill?"). This is documented in a new ## Platform Adaptation bullet.
  • Non-interactive (but main-agent): child skills run with --non-interactive; /grill → written-findings, /diagram → --on-failure drop, /polish → mechanical-only; persona reconciliation AUTO-PICKs + buffers OQ.

4. New-phase contracts (→ reference files)

Per progressive disclosure (§structure), each new phase's detailed contract lives in a reference/ file; SKILL.md carries the gate + a one-paragraph summary + the citation.

4.1 Research phase — reference/research-phase.md (F3)

  • Warrant check: skip when the doc is internal/template-backed AND gathered_context already covers the template's preconditions. Log research: skipped (sufficient context) or research: warranted (<reason>).
  • Plan: propose 3–6 research questions/areas; AskUserQuestion approve/edit (Recommended = Approve). Non-interactive AUTO-PICKs.
  • Fan-out: general-purpose subagents, one per area, strict output contract (stable idx, verifiable URLs, confidence field, "omit rather than guess"); validating+tolerant merge (per the /ideate music-coach learning). Concurrency cap ~4.
  • Output: <feature_folder>/research/<slug>-research.md — loose sidecar (OQ2), plain markdown, source-attributed; NOT indexed/commentable. The draft reads it as tagged gathered_context.

4.2 Persona panel — reference/persona-panel.md (F4)

  • Persona source: template frontmatter personas: list if present; else recommend 3–4 from doc type + context, confirm via one AskUserQuestion.
  • Dispatch: one sonnet (OQ3) general-purpose subagent per persona, parallel (cap 4). Each gets the chrome-stripped draft + a persona brief (role, what they care about, what would make them reject the doc) + the _shared/reviewer-protocol.md contract (≥40-char verbatim quote per finding; section ids from sections.json).
  • Validation: parent-side per reviewer-protocol (quote substring-grep, section-id set check). Invalid finding → dropped.
  • Reconcile: present findings per _shared/findings-dispositions.md (≤4/batch, four dispositions) BEFORE editing the draft. Apply approved via Edit; re-emit sections.json. NFR-1: log a one-line "panel surfaced N findings the eval reviewer did not" so duplication is visible.

4.3 Diagram pass — reference/diagram-pass.md (F5)

  • Propose: 1–3 candidate diagrams (type + clarified concept + target section id). AskUserQuestion multiSelect which to generate (or none). Recommended = the single highest-value one.
  • Generate: per approved diagram, a /diagram subagent — but since subagents can't invoke /diagram, the main agent invokes /diagram --non-interactive --on-failure drop --out <feature_folder>/assets/diagrams/<n>.svg --source <draft excerpt> sequentially (or via the Skill tool). Mandate the dark-mode background <rect> in the diagram brief.
  • Validate before insert: SVG parses; has the bg-rect; after inline insertion at the target section anchor, re-run the heading-id + build_sections_json.js + comments-coverage smoke. Any failure → skip that diagram, log diagram: dropped <n> (<reason>), leave the draft unchanged.
  • Preference: after a successful pass, offer "always run the diagram pass for this project?" → persist .pmos/settings.yaml :: artifact.diagram_pass: true. When set, the pass runs at standard too (still propose+approve).

4.4 New-template guidelines — reference/new-template-guidelines.md (F1/F5-support)

Generic best practices the proposal flow + save-checklist read: 8–15 sections with single-purpose each; eval items split precondition/judgment; a sensible personas: set; a length_target; a default_preset. The save offer (FR-4) is gated on this checklist passing; default = No.

5. /polish + /grill specifics

  • /polish (3.8): main agent calls Skill /polish <artifact> [--non-interactive] on the saved HTML. /polish round-trips HTML and auto-applies mechanical fixes; its existing per-finding surface handles voice-risk. /artifact just invokes it and reports the polish summary. No new logic beyond the invocation + the subagent-degradation note.
  • /grill (3.9): main agent calls Skill /grill <artifact> --depth <brief|standard|deep mapped from --depth> [--non-interactive]. Interactive → turn-by-turn; non-interactive → written findings. Grill's findings are surfaced; high-severity ones offered as edits via findings-dispositions. --no-stress-test-style escape NOT added (grill is mandatory per F7); but a brief depth keeps it to one light pass.
