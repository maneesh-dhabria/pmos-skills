What /artifact already owns vs. net-new

Key finding from reading the current 426-line SKILL.md: the create flow (Phase 2, 8 steps) already owns context-gathering, template resolution, gap-interview, draft generation, and a single reviewer-subagent eval loop (Phase 3, 2-loop cap). Separately, /artifact template add already has a research-grounded authoring flow (T.1–T.6) with a research subagent, section-by-section alignment, eval-gen, save-to-~/.pmos/artifacts/templates/, and an optional dry-run. Two brief capabilities therefore shrink from "build" to "wire-up/extend."
