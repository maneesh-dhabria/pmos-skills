/learn-list — Requirements Tier 3

Mode: skill-new · Plugin: pmos-learnkit · Seed: docs/pmos/ideate/2026-06-03_learn-list.html · Acceptance criteria: the produced skill must conform to reference/skill-patterns.md §A–§F.

Problem

Ramping on an unfamiliar topic means drowning in SEO/AI slop and page-1 search noise, not knowing who the real experts are, and assembling a reading list by hand over hours. Google/Perplexity optimize for an answer, not a learning map; "Awesome" lists and syllabi are static, single-curator, and unevenly fresh.

Solution direction

/learn-list <topic> produces a verified, anti-slop, multi-format curated reading list organized by a canon-derived topic outline. Each link is ranked within its topic and annotated with a ≤2-sentence (~240-char) "why included." It closes with a follow-list (people · newsletters · books-with-summaries · practitioners' signature writings · podcasts) and a copy-ready paste-block. Effort scales by mode (quick/standard/deep).

Load-bearing architectural commitment (from the brief's H×H risk thread): this is a verification-first web pipeline, not a generate-from-memory one. Every link is fetched/verified before emit; the canon is found via live search. Get this wrong and nothing downstream matters.

User journeys

  1. Cold ramp. "/learn-list prompt caching" → outline + ranked annotated links per topic + follow-list. User opens ≥5 links, subscribes to ≥3 sources.
  2. Deep dive. "/learn-list mechanistic interpretability --mode deep --level practitioner" → wider fan-out, 2-hop adjacencies, book summaries + signature writings, practitioner-graph sourcing.
  3. Quick scan. "/learn-list --mode quick" → 3–5 topics, top-3 links each, no adjacencies; sub-2-minute.
  4. Hand-off. User copies the paste-block into Readwise/Notion; optionally runs /primer <topic> for a taught explainer of the same topic.

Non-functional

  • Anti-slop is the differentiator — a source-tier rubric (primary > practitioner > reputable publication > aggregator), named-author requirement, recency awareness. Not vibes.
  • Cost guard — fan-out width and adjacency hops capped per mode; topics deduped before fan-out; deep mode warns on cost.
  • Standalone utility — does NOT load workstream context (like /primer, /diagram). Lives outside requirements→spec→plan.
  • Reuse — learnkit _shared/ (pipeline-setup, html-authoring, social-sourcing free-fetch ladder, resolve-input, non-interactive, interactive-prompts).

Out of scope (v1)

  • Living / re-runnable dedup-against-saved (needs persistent state) — deferred.
  • Direct one-click subscribe / Readwise API export — the paste-block is the integration surface.
  • Sequenced learning-path with done-checks / quizzing (7★+ in the brief) — out; this ships the 6★ annotated list.
  • Inline book/essay full summaries generated by fetching whole books — only sourced existing summaries.

Success signals

Leading: <1 dead/wrong link per 20 emitted; ≥5 links opened per run. Lagging: beats ≥2h of manual searching; ≥3 source-follows within a week.
