Architecture & phase flow

Single SKILL.md (~≤500 body lines) + 4 reference files + 1 HTML artifact template. Mirrors /primer's structure and reuses learnkit _shared/. No bundled executable scripts (verification is done via WebFetch inline) → skill-eval group 10.E skipped by design. Standalone utility: does NOT load workstream.

PhaseDoes
0 · Setup + Load LearningsInline _shared/pipeline-setup.md; resolve {docs_path}, {learn_list_dir}={docs_path}/learn-list/; resolve output_format, mode (interactive/non-interactive); read ~/.pmos/learnings.md entries under ## /learn-list.
1 · IntakeParse <topic>, --mode (default standard), --level (ask once if absent; skip in quick). Consolidated confirm.
2 · Canon & curations (live search)Find 3–5 canonical books, top practitioners, 2–4 existing curations/Top-N lists/awesome-repos. Verify each exists/active. Never from memory.
3 · Outline (cascade + provenance)Derive taxonomy: ≥2 sources agree → use; else curation consensus; else model best-effort tagged provisional. Dedupe topics. Confirm/edit gate in standard+deep; auto in quick (G4).
4 · Source per topic (fan-out)One subagent per topic (mode-gated width). Each: search candidates → anti-slop rank (source-tiers.md) → fetch-verify only the survivors (G1) → keep those that are reachable + real + annotation-grounded (G2). Return verified, ranked, annotated links.
5 · Adjacencies + follow-listAdjacency walk (hops by mode); separate "adjacent rabbit holes" section. Assemble follow-list: people, newsletters, podcasts, books-with-summaries, signature writings (search-consensus, no fake metrics).
6 · Eval + WriteReviewer pass: dead-link sweep, slop spot-check, coverage, annotation-grounding + ≤240-char. Emit HTML + paste-block (+ .md sidecar if --format both). Print path + /primer handoff.
7 · Capture LearningsNumbered phase; append to ~/.pmos/learnings.md under ## /learn-list or emit the "no new learnings" line.

Functional requirements

FRRequirement
FR-1Frontmatter: name: learn-list (== dir), single-line quoted description (≤1024) carrying what + when + ≥5 user-spoken trigger phrases + assertive voice, user-invocable: true, argument-hint enumerating every parsed token. (§A,§B,§F)
FR-2--mode quick|standard|deep drives the dial matrix in reference/modes.md (topics, links/topic, adjacency hops, fan-out, follow-list depth). Default standard.
FR-3--level beginner|practitioner shapes sourcing; if absent ask once in intake (skipped in quick → level-neutral).
FR-4Verification-first (load-bearing): no URL is emitted unless fetch-verified this run. Unverifiable links are dropped, never guessed/recalled. (Anti-pattern #1.)
FR-5Rank-then-verify-survivors (G1): verify only the post-ranking top-N that will be emitted, not every candidate. Verification spend scales with output.
FR-6Verified pass-bar (G2): reachable (not 404/parking/login-wall-without-alt) AND identity-matches the named content AND the ≤2-sentence annotation is grounded in the fetched content. Fail any → drop.
FR-7Anti-slop rubric: reference/source-tiers.md — hard gate (named author OR recognized publication/practitioner) + tier ranking (primary > practitioner > reputable pub > aggregator) + recency awareness.
FR-8Outline cascade + provenance (G3): ≥2-source agreement → curation consensus → provisional model outline; always label where the outline came from.
FR-9Outline-confirm gate (G4): show outline, allow add/drop/reorder before fan-out in standard+deep; auto-proceed in quick and non-interactive.
FR-10No fixed per-format quota; include only formats actually found+verified. Fabricating a source is a hard failure. (D7)
FR-11Each link: rank within topic + ≤2-sentence, ≤~240-char "why included" annotation (D5).
FR-12Paywalled canonical sources: include with a paywalled tag + free alternative when one exists (D3).
FR-13Follow-list: people · newsletters · podcasts · books-with-(sourced)-summaries · practitioners' signature writings (search-consensus, no fabricated citation counts — D2).
FR-14Adjacency walk hops by mode (0/1/2); output in a separate "adjacent rabbit holes" section so it never dilutes the core.
FR-15Output: HTML artifact at {learn_list_dir}/{YYYY-MM-DD}_<slug>.html via the learnkit html-authoring substrate, PLUS a copy-ready markdown paste-block of all links. --format both → full .md sidecar.
FR-16Closing chat: print artifact path + suggest /primer <topic> handoff (standalone — no auto-invoke, D4).
FR-17Cost guard: fan-out width + adjacency capped per mode; topics deduped before fan-out; deep mode logs an est-cost note.
FR-18Standalone: do NOT load workstream context. Body has ## Platform Adaptation, ## Track Progress, learnings load-line, numbered ## Phase 7: Capture Learnings.
FR-19Non-interactive contract: degrade gates per _shared/non-interactive.md (Recommended → auto-pick; outline gate auto-proceeds).
