Problem statement

Product managers make high-stakes judgment calls under ambiguity, but have no low-friction, repeatable way to practice the underlying critical-thinking muscles or to see whether they are improving. Reading about biases and frameworks does not transfer to better reasoning; deliberate, varied reps with concrete feedback do. Existing tools (PM interview prep, generic forecasting trainers) are one-off, human-graded, or off-domain.

HMW: How might we give PMs a low-friction, repeatable way to practice and measurably improve critical thinking — sized to the minutes they have, scoped to problems they actually face, and forcing them to reason in their own words — without it feeling like homework?

Target users & JTBD

Who: Product managers and product builders (IC → senior); anyone making product judgment calls under ambiguity.

When/where: a spare 5–30 minutes in a terminal/CLI session, often inside a working repo; manually triggered.

JTBD: "When I've got a few spare minutes — or I'm staring at a hard product decision — help me sharpen my reasoning under ambiguity and show me I'm getting better, without it feeling like a test."

Goals & non-goals

Goals: (1) a manually-triggered practice session sized to a user-chosen time band; (2) a varied mix of PM-scoped reasoning exercises, each with its own generation + evaluation spec; (3) own-words reasoning graded on moves, not the answer; (4) an objective, accumulating progress signal (calibration + per-muscle); (5) optional repo-contextual scenarios without forcing it.

Non-goals: not an app/server (local files only); not a human-graded course; not a general-knowledge quiz; no auth/accounts; no backend; v1 excludes Socratic, steelman-then-break, red-team-a-PRD, decision-journal, and compare-to-expert shapes (deferred to v2 per the brief).

User journeys

Journey 1 — Quick rep (5–10 min)

User runs /critical-thinking → picks "Quick" band → skill assembles 2–3 mixed exercises (e.g. spot-the-bias + pick-and-defend) → user answers in their own words / picks an option → skill grades the reasoning moves, names one thing missing → session summary shows muscles touched + calibration nudge.

Journey 2 — Deep session in a repo (~30 min)

User runs /critical-thinking inside a product repo → picks "Deep" → skill derives 1–2 repo-contextual scenarios (if a clean product-decision framing exists) plus generic ones → 6–8 exercises across all 9 muscles → richer feedback + calibration prediction logged → summary highlights weakest muscle and updates the scorecard.

Journey 3 — Bring your own dilemma

User pastes a real decision they're stuck on → skill runs the moves rubric live (assumptions surfaced? alternatives steelmanned? 2nd-order effects? falsifiability?) → returns structured critique + the questions to resolve before deciding.

Functional requirements

IDRequirement
FR-1Invocation. Manually triggered as /critical-thinking (and /pmos-learnkit:critical-thinking). Standalone utility — does NOT load workstream context. Announces at start.
FR-2Time-band selection. First step: an AskUserQuestion offering 4 commitment bands → exercise counts: Quick 5–10 min (2–3), Standard 15–20 min (4–5), Deep ~30 min (6–8), Marathon 45 min+ (open — keep going until the user stops). Bands proposed; user picks.
FR-3Exercise library. 9 shapes for v1, each a spec with two halves — scenario generation + evaluation/probe — stored in reference/exercise-shapes.md. Core: pick-and-defend, hard-mode open response, bring-your-own-dilemma. Targeted: assumption-hunt, spot-the-bias, what-would-change-your-mind. Analysis: calibration/forecasting, second-order consequence map, reframe-the-question.
FR-4Mix engine. Each session draws a varied set (no shape repeated until others are used); deterministic-with-rotation selector weighted toward under-practiced muscles (read from the scorecard when present). Selection logic specified in the body; no two consecutive exercises share a shape.
FR-5PM problem scope. Scenarios drawn only from PM/product-builder domains — product design, prioritization & tradeoffs, metrics & experimentation, influence & stakeholder management, strategy under ambiguity, GTM. Scenarios are generated fresh at runtime from these domains (no static template bank) so they are believable and varied, not blank-page pablum.
FR-6Own-words first. Free-form answers are the default, each with explicit instructions on the expected answer shape/length. AskUserQuestion is used ONLY for genuinely multiple-choice shapes (pick-and-defend option select, spot-the-bias). When AskUserQuestion is unavailable, degrade to numbered free-form prompts with clear answer instructions.
FR-7Move-based grading. Feedback evaluates the reasoning moves per the shape's rubric (e.g. assumption surfaced? alternative steelmanned? 2nd-order effect traced? falsifiable claim stated?), NOT the choice. Every graded answer MUST name at least one concrete thing that was missing or weak — never pure praise.
FR-8Repo-aware (don't force it). When run inside a git repo, generate 1–2 exercises from repo context only if a clean product-decision framing is derivable; otherwise stay generic. Every session keeps ≥1–2 generic (non-repo) problems regardless. Never quote secrets/credentials; cap repo exercises at 2.
FR-9Calibration scoring. The calibration shape elicits a probability estimate + reasoning; the skill records prediction/outcome pairs and computes an accumulating Brier score across sessions — the objective "you're improving" signal.
FR-10Scorecard persistence. A per-session scorecard persists locally under ~/.pmos/learnkit/critical-thinking/ (created on first run): per-muscle scores, weakest muscle, calibration history, and shape history (to drive FR-4 rotation). Format documented; absence handled gracefully (first run seeds it).
FR-11Session flow + summary. intro → N exercises (generate → user answers → grade/probe) → session summary (muscles practiced, per-muscle takeaways, weakest muscle, calibration trend, optional streak). Summary written to chat; scorecard updated atomically.
FR-12Platform Adaptation. Section covering AskUserQuestion degradation (FR-6), no-subagent operation (single-agent), and that the skill needs no .pmos/settings.yaml (standalone) — but follows pmos skill conventions.
FR-13Capture Learnings. End-of-skill learnings reflection emitting exactly one line (learning written, or "no new learnings because <reason>").
FR-14Anti-Patterns section. Documents the failure modes from the brief's premortem (mushy praise grading; blank-page unbelievable scenarios; forcing repo context; grading the choice not the reasoning; repeating the same 2 shapes).
