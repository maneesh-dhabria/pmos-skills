Gaps Surfaced (8) — all addressed

  1. CLAUDE.md cutover is 5 sections, not 1. The prior revision implied a single new ## Release policy section. Live inspection found 4 existing sections (## Canonical skill path, ## Skill-authoring conventions, ## Plugin manifest version sync, ## Release entry point) that all reference single-plugin plugins/pmos-toolkit/ paths and must be generalized in the same cutover. → D18 added.
  2. "NOT modifying /complete-dev" Non-Goal was unachievable. Live inspection of complete-dev/SKILL.md found 10+ hardcoded plugins/pmos-toolkit/ references. → Q3 flipped this; D11 reworded; D6 stays dropped (no separate release script needed — /complete-dev does the job).
  3. Pre-push hook validation was 2-way, must be 3-way. Live inspection of .claude-plugin/marketplace.json showed plugins[].version = "1.0.0" (stale, never bumped, while plugin.json is at 2.49.0). CC uses this field as the canonical update signal. → D12 amended to 3-way; D17b added.
  4. D16 was over-engineered (triple-layer). Pre-commit + pre-push + CI was belt-suspenders-airbag for a solo project. → Q4 collapsed to pre-commit only.
  5. J2.5 hotfix journey was wrong as written. Tag on a hotfix branch doesn't reach end-users via marketplace install (CC reads marketplace.json at HEAD-of-main). → Q2 removed J2.5 entirely; D10 collapsed to forward-only.
  6. D13 / D15 README-banner language contradicted Q1. Archive + private means no public banner. → D13 amended; D15 rewritten ("README of new repo only"); OQ11 closed; OQ13 added then closed.
  7. Marketplace name field change unstated. The marketplace's top-level name field is currently "pmos-toolkit"; must flip to "pmos-skills" at cutover. Also affects CC's plugin cache directory layout (~/.claude-personal/plugins/cache/pmos-toolkit/... → ~/.claude-personal/plugins/cache/pmos-skills/...). → D17 added.
  8. Mixed pre-cutover tag conventions undocumented. Live inspection: three formats (v2.x, pmos-toolkit-v2.x, pmos-toolkit-2.x). → Q5 + D7 (revised) made the "preserve, don't rewrite" choice explicit.
