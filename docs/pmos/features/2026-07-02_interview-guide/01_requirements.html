/interview-guide — requirements (epic 260702-3bb, route:skill)

Light epic-level framing for a new-skill define. The detailed decisions, invariants, output shapes, and story map are in 02_design.html (the coherence contract). This doc frames the problem and the shared shape only.

Problem

A hiring manager preparing an interview round has to author, from scratch each time: a guide for the interviewer (how to run the round, what to probe, what strong looks like), a scoring sheet, and — for case rounds — a candidate-facing case document. The research for "what a good PM round assesses" already exists inside /interview-feedback (7 researched round archetypes) but is trapped in that skill and only used after the interview, for scoring. There is no tool to author the guide before the round.

Solution direction

  • A new pmos-managerkit skill /interview-guide that authors three artifacts per round — interviewer reference, scoring sheet, and (case rounds only) a candidate-facing case document.
  • Two input modes: from the round's stated requirements/competencies, or from best-practices for a chosen PM round archetype.
  • Grounded in the existing 7-archetype guidelines corpus, first lifted into a shared managerkit substrate that both /interview-guide and /interview-feedback consume.
  • Output interoperates with /interview-feedback: the generated scoring sheet carries the same machine anchors, so a candidate can later be scored against the very guide authored here.

Inputs needed from the user

  • Role being hired for.
  • Expected skills / competencies to be assessed in the round.
  • Round type (one of the 7 PM archetypes, or custom).
  • For case rounds: a business context / domain (or a rough case seed).

Acceptance shape

  • The guidelines corpus is abstracted to a shared reference, consumed by both skills, with /interview-feedback behavior unchanged (INV-1).
  • /interview-guide produces the three artifacts (case doc only for case rounds), grounded in the corpus, with a self-review pass; the manager reviews/edits (D6).
  • Conforms to skill-patterns.md §A–§L; passes skill-eval; hygiene lints green (INV-4).

Auto-tier: Tier 2 (a new skill with a substrate extraction and interop contract; not a one-file bug fix, not a large multi-plugin platform change).
