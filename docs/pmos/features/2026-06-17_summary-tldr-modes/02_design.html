Design — /summary-tldr output modes + /diagram mindmap capability

Epic 260617-jy8 · route: skill · plugin: pmos-toolkit · 2026-06-17 · This is the epic design_doc — the cross-skill coherence contract; per-story /plans cite its anchors.

Context

/summary-tldr today emits exactly one shape of output: a grounded TEXT TL;DR (Phase 6 emit), with a --style bullets|exec|nested|layered sub-style dimension and an optional Phase 7 hand-off to /diagram for a single add-on diagram. The maintainer wants a new output-mode dimension — --mode narrative|mindmap|video|shorts (also an interactive picker) — so the same grounded source can be rendered as a mindmap tree, an explainer video, or a swipeable card carousel, in addition to the canonical text.

Four modes:

  • narrative — the default; today's grounded text TL;DR (unchanged, back-compat).
  • mindmap — a tree/radial diagram of the source's key arguments and takeaways.
  • video — a narrated slideshow .mp4 via /explainer-video.
  • shorts — a swipeable carousel of bite-size (≤140-char) cards, one key takeaway each, each pairing relevant media when one exists.

Define-time spike — /diagram cannot do mindmaps today

A read-only spike this run (against plugins/pmos-toolkit/skills/diagram/) found /diagram is insufficient for mindmaps: it lists mental-model in its description and picks a 1:1 radial canvas, but it has no automatic layout engine — every node is hand-placed by coordinate (SKILL.md Phase 3, svg-primer.md) — and hard-fails above 30 nodes (eval/code-metrics.md; the 21–30 split prompt at SKILL.md ~line 178). Grep across the skill found no mermaid/dagre/elk/d3/auto-layout. Hand-placing a 15–30-node radial tree reliably triggers overlap/tunnel hard-fails. So the mindmap mode requires a new /diagram capability — making this a two-skill epic.

Goal & non-goals

Goal. (1) Add the --mode output dimension to /summary-tldr — narrative default (byte-for-byte back-compat) plus mindmap/video/shorts renderings derived from the grounded original source. (2) Add a new --mode mindmap to /diagram backed by a vendored, zero-dependency auto-layout, with /diagram as the post-layout theme + eval engine.

Non-goals. Per-card video generation (shorts pairs existing media only); multi-mode-per-run (single mode per run in v1); replacing the canonical text summary (it is always emitted); any cloud / browser-render dependency (mermaid + puppeteer rejected, see D5); cross-document synthesis; re-hosting the video (it is linked, not copied). Writing curator commentary is N/A here.

Decisions

#Decision
D1--mode is a NEW output-format dimension, orthogonal to --style. Default narrative. --style shapes only the narrative text; with a non-narrative mode it is recorded as ignored with a one-line warn (the canonical text still emits at the default/lastrun style).
D2The grounded TEXT summary is ALWAYS emitted first as the canonical, crash-safe artifact, in every mode. Non-narrative renderings are produced in addition, after the canonical emit — the same emit-before-slow-step discipline as today's Phase 6→7 (the approved text is persisted by construction before any slow rendering runs).
D3Non-narrative renderings derive from the grounded this-run extraction of the original doc (the cleaned full text + keyfact list from Phase 4), NOT the lossy compressed prose. Mindmap & shorts derive from the keyfact/structure extraction; video re-ingests the ORIGINAL SOURCE via /explainer-video (its own grounded ingest). All existing grounding rules hold — nothing from model memory (INV4).
D4Mindmap mode delegates to /diagram --mode mindmap (the new capability). /summary-tldr never builds a renderer (INV5).
D5/diagram --mode mindmap uses a vendored, zero-dependency tidy-tree/radial layout module (Reingold–Tilford / Buchheim linear-time + a radial depth→radius, breadth→angle transform) to compute coordinates, then theme-authors SVG with existing theme tokens. No npm, no npx, no network, no browser render-path for layout. npx rejected: d3-hierarchy ships no CLI bin, so npx -p d3-hierarchy … fetches the package over the network at runtime — breaking the offline + deterministic posture /diagram and /explainer-video share. mermaid rejected: needs puppeteer/a browser and clashes with the theme tokens. Layout is pure math → deterministic (no Math.random/Date; resume-safe).
D6Mindmap mode relaxes /diagram's >30-node hard cap (auto-layout handles density) up to a documented ceiling (~60); the overlap / tunnel / legibility / angular-resolution eval still applies. A TL;DR keyfact list naturally bounds nodes (~10–25).
D7Shorts = a swipeable carousel of ≤140-char cards, one key takeaway each (BLUF), emitted as a self-contained SIBLING artifact <slug>-shorts.html (swipe + keyboard nav, zero-dep, offline), linked from the canonical summary + library. Each card pairs relevant existing media when available; else full text. NO per-card video generation.
D8Shorts media comes from the source's figure inventory (reuse /explainer-video's ingest.mjs deterministic figure extraction) plus any artifact this run already produced (the mindmap SVG, the video). Relevance = keyfact ↔ figure (alt / caption / proximity). No fabrication; a card with no relevant media is text-only.
D9Video mode delegates to /explainer-video on the ORIGINAL SOURCE (main-agent hand-off — skills cannot invoke skills from a subagent). Length maps from --compression (tight→quick, standard→standard, detailed→deep); a --video-length quick|standard|deep override is honored. The .mp4 lands in /explainer-video's dir; /summary-tldr links it + records provenance (not re-hosted).
D10Mode-selection UX: interactive with no --mode → an AskUserQuestion picker with Narrative (Recommended); --mode <m> pre-answers and skips it; non-interactive AUTO-PICKs narrative. Single mode per run (v1; a comma-list is a deferred future).
D11Graceful degradation (additive modes + D2 make this safe): a mode whose deps/inputs are unavailable (no ffmpeg/Playwright/TTS for video; too few keyfacts for a mindmap; <2 cards for shorts) reports the precise reason + an install/why hint and STILL ships the canonical text summary. Never fabricate; never silent-fail (mirrors the skill's existing honest-degradation rule).
D12Mode × source-kind feasibility floors: mindmap needs ≥ a handful of keyfacts; video defers slide-count calibration to /explainer-video; shorts needs ≥2 cards. Below floor → degrade-with-note per D11.

Invariants (cross-skill coherence contract)

  • INV1 — /diagram --mode mindmap is a self-contained, independently-shippable, independently skill-eval'd capability. /summary-tldr's mindmap mode depends on it (story dep; claim-time transitive merge makes it present in the consumer's worktree before its build, per build-loop D9).
  • INV2 — --mode (output format) ⟂ --style (narrative sub-style). The canonical text emits at the resolved narrative style; --style is meaningful only when mode==narrative, else ignored-with-warn (D1).
  • INV3 — the canonical grounded text summary is emitted first (crash-safety) before any mode rendering runs (D2).
  • INV4 — every rendering traces to the grounded this-run extraction / original source; the Phase 5 first-time-reader review + grounding rules hold; nothing from model memory (D3).
  • INV5 — reuse, don't rebuild: mindmap → /diagram --mode mindmap; video → /explainer-video; shorts media → ingest.mjs figure inventory. Zero duplicated renderers inside /summary-tldr.
  • INV6 — additive + back-compat: mode==narrative reproduces today's behavior exactly; every new flag is optional; existing /summary-tldr and /diagram tests stay green.

Functional requirements

Story A — /diagram --mode mindmap (story 260617-1aq, no deps)

  • FR-A1 — new --mode mindmap value (alongside diagram/infographic) parsed in Phase 0; it accepts a hierarchical input (root + nested nodes) from --source and/or the description; a flat description degrades to a single-level radial.
  • FR-A2 — a vendored zero-dep layout module (e.g. scripts/mindmap-layout.mjs) computing tidy-tree + radial coordinates as pure functions, with a --selftest over fixtures; deterministic (no Math.random/Date) per D5.
  • FR-A3 — a mindmap drawing path that authors themed SVG from the computed coordinates using the active theme's tokens (palette/typography/strokes, curved connectors for tree/radial edges per the theme's connector rules), and writes the sidecar with layoutEngine + computed positions.
  • FR-A4 — node-cap relaxed for mindmap mode (>30 up to a documented ceiling, D6); overlap/tunnel/legibility/angular-resolution eval still applies (no weakening of the gate).
  • FR-A5 — non-interactive contract preserved: optional --approach (e.g. radial vs tree), --on-failure drop|ship-with-warning|exit-nonzero honored, returns an SVG a caller can inject; exit-code contract unchanged.
  • FR-A6 — fixture-based layout selftest + an eval/apply test; passes skill-eval.md [D]; conforms to skill-patterns.md §A–§L + host CLAUDE.md.

Story B — /summary-tldr mode scaffold + narrative refactor + mindmap mode (story 260617-xn4, deps 260617-1aq)

  • FR-B1 — parse --mode narrative|mindmap|video|shorts (default narrative); interactive picker (D10) with Narrative Recommended; --mode pre-answers; non-interactive AUTO-PICK narrative; invalid value → platform-aware error naming the set; exit 64.
  • FR-B2 — refactor current behavior as mode==narrative — byte-for-byte back-compat (INV6); --style applies only here, else ignored-with-warn (D1/INV2).
  • FR-B3 — the canonical grounded text summary is always emitted first (D2/INV3), regardless of mode, before any rendering.
  • FR-B4 — mindmap mode derives a hierarchy from the keyfact extraction (root = topic; branches = key arguments; leaves = takeaways/numbers, D3), then main-agent hand-off to /diagram --mode mindmap --non-interactive --on-failure drop; validate the returned SVG (parses, theme background, heading-id smoke) before saving as a sibling <slug>-mindmap.svg and linking it from the canonical doc + library (same handoff+validate pattern as today's Phase 7).
  • FR-B5 — graceful degradation (D11/D12): too few keyfacts → note + text-only canonical ships.
  • FR-B6 — tests + skill-eval; existing /summary-tldr tests stay green (INV6).

Story C — /summary-tldr --mode video (story 260617-gfx, deps 260617-xn4)

  • FR-C1 — video mode: main-agent hand-off to /explainer-video on the original source (D9); length mapped from --compression with a --video-length override.
  • FR-C2 — link the produced .mp4 + provenance into the canonical doc + library; do NOT re-host.
  • FR-C3 — graceful dep degradation (no ffmpeg/Playwright/TTS) → the precise install hint + the canonical text still ships (D11).
  • FR-C4 — tests + skill-eval.

Story D — /summary-tldr --mode shorts (story 260617-wf6, deps 260617-xn4)

  • FR-D1 — derive ≤140-char cards (one key takeaway each, BLUF) from the keyfact extraction (D3/D7); ≥2-card floor (D12).
  • FR-D2 — emit a self-contained sibling <slug>-shorts.html: a swipeable + keyboard-nav carousel, zero-dep, offline (no CDN), compatible with the inline-comments overlay; linked from the canonical doc + library.
  • FR-D3 — media pairing (D8): reuse ingest.mjs figure inventory + any mindmap/video produced this run; relevance match keyfact↔figure; text-only card when no relevant media; never fabricate.
  • FR-D4 — graceful degradation + tests + skill-eval.

Story split

Four stories on two skills in one plugin (pmos-toolkit), a clean DAG:

260617-1aq  (/diagram --mode mindmap)        no deps      ─┐
260617-xn4  (/summary-tldr scaffold+narrative+mindmap)  deps: 1aq  ─┐
260617-gfx  (/summary-tldr --mode video)     deps: xn4   ─┤
260617-wf6  (/summary-tldr --mode shorts)    deps: xn4   ─┘
  • 1aq is a standalone /diagram capability that B's mindmap mode needs (INV1; claim-time transitive merge makes it present before B builds).
  • xn4 is the mode-dispatch scaffold + narrative back-compat + the cheapest new mode (mindmap, once 1aq exists) — the first end-to-end "modes work" vertical slice.
  • gfx and wf6 are independent leaf modes, each its own shippable slice, both depending only on xn4's scaffold. The D24 litmus holds — their tasks never cross each other (the only cross-story dep is the story-level "scaffold exists").

All four are route: skill, plugin pmos-toolkit (single-plugin, D17 satisfied). Each is scored once against skill-eval.md.

RiskMitigation
/explainer-video is heavy (ffmpeg/Playwright/TTS) and may be absent on a host.D11 graceful degradation + D2 always-emit canonical text — video mode never blocks the run.
Vendored layout math could be subtly wrong at scale.Fixture selftests (FR-A2) + the existing /diagram overlap/tunnel/angular eval gate (FR-A4) catch bad layouts.
Shorts carousel JS could clash with the inline-comments overlay.FR-D2 explicitly requires offline, zero-dep, comments-overlay-compatible; verify at build (live dogfood).
Scope (4 stories).Critical path is 1aq→xn4 (mindmap end-to-end); gfx/wf6 are independent leaves that can ship incrementally if needed.
Back-compat regression in /summary-tldr.INV6 + FR-B2 require mode==narrative to reproduce today exactly; existing tests stay green as an AC.
