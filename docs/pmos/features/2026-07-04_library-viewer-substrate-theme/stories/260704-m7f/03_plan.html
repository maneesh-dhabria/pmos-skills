Plan — re-theme the library-viewer substrate

story 260704-m7f · epic 260704-vde · route: skill · pmos-learnkit · plan scoped by 02_design.html §3–§4 anchors + this story's ACs. Machine-readable task queue: tasks.yaml.

Goal

Make _shared/library-viewer/lib.mjs emit browse pages that read as the canonical "Editorial Technical" substrate — by consuming the canonical style.css rather than carrying a private dark theme. All three consumers (/frameworks, /primer, /learn-list) inherit the change for free because they call emitHtml() and never hand-author CSS.

Approach

  • Base layer (T1) — read ../html-authoring/assets/style.css via import.meta.url at build time and inline it first in the <style> block; fail loud if absent (design D5/D6). Stays offline — inlined, no <link> (INV-1).
  • Component layer (T2) — delete the old dark :root block; rewrite masthead, layout, cards, facets, chips, search, reader, iframe reader onto --pmos-* tokens only. Dark mode comes free from style.css's prefers-color-scheme overrides — add no viewer-specific dark rules (INV-3). No class/DOM change (INV-6).
  • Consumer audit (T3) — reconcile any old-token CSS the three build-library.mjs pass via extraHead/config.
  • Guards (T4) — update lib.test.mjs + guidelines.md to the new contract; keep selftest.sh (skill-agnostic grep + node --test) green (INV-4).
  • Evidence (T5) — build all three pages, eyeball both color schemes in a browser, fix any contrast regression (the /verify browser-evidence gate).

Risks & mitigations

  • Dark-layout-on-light contrast loss (e.g. invisible card border on warm paper) → T5 live render in both schemes.
  • Hidden old-token consumer CSS → T3 required audit of all three consumers.
  • selftest skill-agnostic grep false-positive on the new style.css read → T4 adjusts the grep only if it misfires; reading a sibling substrate file is not skill-specific (INV-4).
