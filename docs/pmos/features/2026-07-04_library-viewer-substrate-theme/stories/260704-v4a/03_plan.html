Plan — refresh the 61 bundled primer corpus files

story 260704-v4a · epic 260704-vde · route: skill · pmos-learnkit · plan scoped by 02_design.html §5–§6 anchors + this story's ACs. Machine-readable task queue: tasks.yaml.

Goal

Replace the stale "Mono Minimal" inlined CSS in all 61 bundled primer corpus files (primer/data/primers/*.html) with the current "Editorial Technical" style.css, without touching the primer body. Because the bundled primers are deliberately self-contained (CSS inlined, not linked), the fix is a marker-scoped, idempotent in-place CSS swap — and no such script exists today, so this story builds one.

Approach

  • CLI (T1) — new zero-dep retheme-corpus.mjs. Locate each file's inlined substrate <style> block by its header-comment marker (/* pmos-toolkit html-authoring substrate … */), not the first <style>, and replace only that region's CSS with the current style.css. Preserve <main> byte-for-byte (INV-5). Fail loud on a missing marker; idempotent on re-run.
  • Test (T2) — fixture-based: marker replaced, body unchanged, idempotent, missing-marker errors loudly.
  • Apply (T3) — run over all 61, commit refreshed files, spot-check in a browser, confirm region-scoped diff + clean re-run.
