Open / Deferred

  • Mobile / tablet viewing — deferred; YAGNI for solo-laptop operator; revisit if stakeholders review on phones.
  • Browser-tab synchronization — when overlay is open in browser during a live /comments resolve chat session, browser doesn't see staged edits until reload. Acceptable v1; document in skill body.
  • Re-dispatch loop cap (Q25) — pick a value at spec time (lean: 2 re-dispatches before forcing "leave open and skip").

Gaps surfaced

  • Sidecar-follows-artifact-on-rename — when artifact is renamed/moved, sidecar must move atomically. Add convention/tool rule in _shared/html-authoring/conventions.md.
  • Thread-ID generation scheme — lean: short nanoid-style 8-char UUID; pin at spec time.
  • Apply-edit-at-anchor entrypoint surface — every one of 12+ HTML-emitting skills needs this contract documented and tested. Cross-cutting; will dominate build estimate.
  • System-reply failure-mode taxonomy (Q22) — small enum needed: anchor_orphaned / edit_conflicted / agent_judged_infeasible / agent_errored.
  • Bundle size budget for comments.js — every emitted artifact ships with it; cap (≤20KB minified) keeps artifact bloat bounded.
  • Audit of current 12+ skills' HTML output for stable-id coverage — mandated by conventions.md §3 but worth a one-shot grep before the build commits to id-first anchoring.
