/learn-list

ideation · new skill · 2026-06-03 Created using pmos‑toolkit

TL;DR

/learn-list <topic> turns any topic into a verified, anti-slop, multi-format reading list organized by a canon-derived topic outline — each link annotated with a ≤2-sentence "why included" and ranked within its topic — closing with a follow-list of people, newsletters, books (with summaries) and practitioners' most-cited writings. Effort scales via quick / standard / deep modes (fan-out width + adjacency depth).

Recommendation: GO — but only if built as a verification-first web pipeline, not a generate-from-memory one. The two highest risks (dead/hallucinated links, stale canon) collapse into a single architectural commitment: fetch-verify every link and live-search the canon. Get that right and the rest is tuning.

How Might We

How might we turn any topic into a high-signal, source-diverse reading list — anchored on canonical experts and their own taxonomy of the field — with effort the user can dial up or down, while filtering out SEO/AI slop?

Target user & job-to-be-done

Who: A curious operator/PM/engineer ramping fast on an unfamiliar topic (or going deeper on a familiar one).

When / where: Before a meeting, a new scope, a career pivot, or out of pure curiosity — at the point where Google/Perplexity return generic page-1 noise and they don't yet know who the real experts are.

JTBD: "When I'm ramping on a new topic, give me the best content across every format organized sensibly — plus the people and sources to follow next — so I can go from zero to conversant without drowning in slop."

Hypothesis

We believe that building the list canon-first (find the experts, harvest their taxonomy and their own recommendations) and verifying every link before emit will produce a list that demonstrably beats 2 hours of the user's own searching and earns ≥3 real source-follows because the unmet gap isn't "more links" — it's trust, non-obviousness, and curation that search engines and memory-based LLMs structurally cannot provide.

Idea variants considered

Generated via First Principles + Analogous Inspiration. Finalist ingredients flagged with ★.

  1. ★ Canon-first cascade — find 3–5 canonical sources → extract their topic taxonomy → fan out subagents per topic. spine
  2. Practitioner-graph traversal — build the list from who the top practitioners cite/recommend (expert social proof).
  3. ★ Curation-of-curations — harvest existing Top-N lists / syllabi / awesome-repos first, merge & dedupe, then fill gaps.
  4. Question-driven — organize by the key questions a learner must answer (learning-outcome, not topic).
  5. Format-balanced quota — enforce ≥1 of each format per topic. (Rejected — invites fabrication; coverage should be honest.)
  6. Progressive-depth ladder — output as a path (primer→core→frontier→adjacent). (Deferred — heavier than chosen scope.)
  7. Credibility-scored & ranked — rank within topic by author authority/recency. folded into spine
  8. ★ Adjacency as a dial — core list + separate "adjacent rabbit holes"; quick/standard/deep set how far the adjacency walk goes.
  9. ★ Learner-shaped — short upfront level/goal/time signal so a beginner vs. practitioner gets a different list.
  10. ★ Anti-slop filter pass — down-rank SEO/AI listicles, reward primary + practitioner sources. The core differentiator.
  11. Living / re-runnable — dedupe against what you've saved; show only what's new. (Deferred — needs state; out of scope for v1.)
  12. ★ Book-summary + most-cited-writing sourcing — quality summary per canonical book; each practitioner's most-cited piece.

How it works (chosen finalist)

A recommended verification-first pipeline. Modes set the dials at each fan-out point.

  • 0 · Shape — parse topic; one short prompt for level / goal / time (skippable). Resolve mode: quick / standard / deep.
  • 1 · Find the canon (live web search, not memory) — identify 3–5 canonical books, the top practitioners, and 2–4 existing curations (Top-N lists, syllabi, "awesome" repos, expert reading lists). Verify each exists and is current.
  • 2 · Derive the outline — extract the topic taxonomy from the canon + curations (not the model's own guess), so an expert would recognize the structure. Dedupe topics before fan-out.
  • 3 · Fan out per topic (subagents) — each subagent sources high-quality, multi-format content for one topic. Width capped by mode. Every candidate URL is fetched/verified; unverifiable links are dropped, never guessed.
  • 4 · Anti-slop & rank — score each candidate on a source-tier rubric (primary > practitioner > reputable pub > aggregator), named-author requirement, recency. Keep top-ranked per topic; annotate each with ≤2 sentences on why it's in.
  • 5 · Adjacency walk — explore adjacent topics to widen coverage; depth set by mode (quick=none, standard=1 hop, deep=2 hops). Output as a clearly-separated "adjacent rabbit holes" section so it never dilutes the core.
  • 6 · Follow-list + summaries — assemble people / newsletters / books (with sourced summaries) / practitioners' most-cited writings, each with a direct subscribe/source link.
  • 7 · Emit — single HTML artifact: canon-derived outline → ranked annotated links per topic → adjacencies → follow-list. Dead-link sweep before write.

Success signals

Leading: <1 dead/wrong link per 20 emitted; user opens ≥5 links per run; user re-runs on a second topic within the same session.

Lagging: User says it beat ≥2 hours of their own searching; user follows/subscribes to ≥3 recommended sources within a week; /learn-list becomes the default first step before a new scope.
