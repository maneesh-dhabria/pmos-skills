# diagram — review

**Grade:** C+ (well-engineered machinery, but a meaningful rewrite would pay off — the QA loop that is the skill's identity is also what broke its one real substrate consumer, and visual-style rules are stated in three places with conflicting authority claims)
**Size:** SKILL.md 555 lines (526 excluding non-interactive block); references 10 md files / ~1,130 lines (plus 2 theme.yaml + _schema.json + _shared-palette.yaml ≈ 430 lines, and ~3,800 lines of scripts/tests across 20 files); target ~320 lines.

## TL;DR

- **Biggest win available:** make non-interactive mode self-fixing. Today Phase 6's Findings Presentation Protocol is `defer-only: ambiguous` + "do NOT silently self-fix", so headless runs cannot use the refinement loop at all — every imperfect draft goes straight to terminal handling. This is exactly why `/frameworks` probed `/diagram`, found it "infeasible headless", and built a parallel bespoke SVG pipeline for 421 diagrams (D5 execute-time deviation). The grill had verified feasibility *from reading SKILL.md* (`grills/2026-06-07_01-requirements.html` answers D5 "Yes"); execution falsified it. One design change — auto-apply proposed fixes in non-interactive mode, recording dispositions in the sidecar — would have kept that consumer.
- **Biggest risk in current design:** taste and enforcement are interleaved. The vision rubric spends a render + subagent call per loop, but only `legibility` (occlusion/clipping/text overflow) truly needs the raster; `arrowhead-consistency` and legend *presence* are code-derivable from the SVG source, and `style-atom-match` largely re-checks what the palette/typography/contrast hard-fails already enforce deterministically. Meanwhile the `grid_snap ≥ 0.95` soft metric can drag `code_score` below 0.8 and trigger refinement loops over 2px offsets no human can see.
- **Done well, keep:** the deterministic hard-fails (node overlap, edge tunnels, WCAG contrast, 12px font floor, >30 nodes) catch the actual failure modes of LLM-authored SVG and are cheap, stable, and model-version-proof. The exit-code contract (0/2/3/4/64) and `--on-failure` make the skill scriptable. The apply-edit-at-anchor section *cites* the shared contract instead of restating it (per NFR-08) — model substrate behavior other sections should copy.

## Findings

1. **[G][X] Non-interactive mode disables the skill's core value, and it cost a real consumer.** Phase 6 step 2 mandates the Findings Presentation Protocol ("do NOT silently self-fix") and tags it `defer-only: ambiguous`; under `--non-interactive` every finding defers to the OQ buffer and no fix is applied, so the refinement loop is inert exactly when batch callers need it. Separately, Phase 5 hard-binds `high`-rigor to dispatching a `general-purpose` subagent — unavailable when `/diagram` itself runs inside a subagent fan-out (the Platform Adaptation note covers "no subagents" *environments*, not subagent *contexts*). Result: `/frameworks` D5 — requirements pinned the invocation contract `--non-interactive --approach … --on-failure ship-with-warning` (`docs/pmos/features/2026-06-07_frameworks-skill/01_requirements.html`, D5 row), the execute-time probe proved it couldn't run unattended at scale, and 421 SVGs were generated outside the skill. **Fix:** in non-interactive mode, auto-apply the proposed fix for each finding and record `{finding, disposition: auto-applied}` in `evalSummary`; reserve DEFER for re-framing-level choices (node-count split, alt framing). Make reviewer dispatch a runtime capability check ("subagent available? dispatch : inline") instead of a rigor-table binding. The brainstorm interactivity itself is NOT the blocker — `--approach` already bypasses it; flag this in the parent review so the "interactivity requirement" isn't misdiagnosed.
2. **[G] Vision rubric is overweight relative to what only vision can catch.** Of the 7 items: `legibility` is the one genuinely raster-only check (occlusion, clipping, rendered text overflow — code can't measure rendered text width; svg-primer.md only offers a `0.55 × char_count` estimate). `arrowhead-consistency` is derivable from SVG source (mix of `marker-end` present/absent across connectors — the defect fixture `tests/defects/arrowhead-inconsistent.svg` could be caught deterministically). Legend *presence* when ≥2 categorical colors is code-derivable (the legend class is already excluded from node detection, so the evaluator sees it). `style-atom-match` mostly re-judges palette/stroke/radii/type-scale, all of which the deterministic checks already hard-fail — what's left is gestalt taste. `primary-emphasis` and `clear-entry` are design opinions with flake-prone edge cases ("auto-pass if declared in sidecar `approach`"). **Fix:** gating vision set = `legibility` + theme `rubricOverrides.add` items; demote `primary-emphasis`/`clear-entry` to advisory alongside `visual-balance`; move arrowhead-mix and legend-presence into `run.py`. Cuts per-loop latency, flake surface, and model-version coupling while keeping every real catch.
3. **[R] Three competing sources of truth for visual style.** `themes/technical/style.md` line 3: "This file is the single source of truth for `/diagram` visual style" — but Phase 0 loads `theme.yaml` (validated against `_schema.json`), and `run.py` derives the palette set from `theme.yaml`, not style.md. The editorial theme correctly inverts authority ("style.md … mirroring this file"). The technical style.md claim is stale wording from before the 2026-05-06 themes feature introduced theme.yaml. On top of that, SKILL.md Phase 3 steps 4–6 restate the tokens (sizes 12/14/16/20, weights 400/600, strokes 1/1.5/2, radii 0/4/8, pinned roles), and the Anti-patterns list restates style.md §5.9. The same palette rule lives in 3–4 places. **Fix:** theme.yaml = machine truth; style.md = rationale + layout guidance, opening line changed to "mirrors theme.yaml"; Phase 3 says "apply the active theme's tokens (theme.yaml; rationale in style.md)" and deletes the per-token restatement (~25 lines).
4. **[V] Anti-patterns section: 19 bullets, roughly half restate enforced rules.** "Do NOT use colors outside the palette" (palette hard-fail), "font sizes below 12px" (hard-fail), "exceed 30 primary nodes" (hard-fail), "mix connectors…" (role-style check + theme contract), "`<image>`/`<foreignObject>`/filters" (style.md §5.9 + matrix/transform guard), "skip the renderer hard-gate" (Phase 0 already refuses). Pocock's pattern: state an anti-pattern only where the model would plausibly deviate AND nothing else catches it. **Keep:** don't copy atoms as templates, minimal patches on extend, don't brainstorm from a hardcoded type list, don't silently dump prose findings. **Delete or fold the rest** (~10 bullets, ~15 lines).
5. **[Ph][R] Ghost phase numbering — insertion without re-thinking.** "Track Progress" lists phases "(0, 1, 2, 3, 4, 5, 6, **6.6**, 7, 8)" — there is no Phase 6.6 in the file; the body has 6a and 6b, and the original spec called the terminal handler "Phase 6.5" (`2026-05-08_update-skills-diagram-on-failure/01_requirements.md`). Phase 7's stdout template still says "item 7 advisory" after the numeric→stable-ID migration (rubric.md's own mapping note). **Fix:** renumber the list, replace "item 7" with `visual-balance`. Structurally: Phase 6a is two-thirds an exit-code table — fold it into a single "Terminal failure + exit codes" section; Phase 6b's 8 steps duplicate `editorial-v1.md`'s spec territory — leave a ~10-line branch pointer ("infographic mode: generate copy → user review → clamp/anchor via `wrapper/` → compose → slim rubric ships-with-warning; spec: editorial-v1.md") and let the reference file carry the detail, Pocock improve-codebase-architecture-style.
6. **[G] `role-style-consistency` hard-fail can't fire when it matters.** `run.py:evaluate()` resolves the sidecar via `svg_path.with_suffix(".diagram.json")` — for the Phase 4 input `<out>.svg.tmp` that yields `<out>.svg.diagram.json`, and the sidecar is only written in Phase 7 anyway. So during the entire draft/refine loop the code check silently returns ok (missing sidecar → `True`), and enforcement falls to the editorial vision add-item only. The hard-fail only ever fires on re-evaluating finalized SVGs (selftest fixtures, extend flow). **Fix:** write a working sidecar (entities/relationships/`_svgId`s exist from Phase 3) before Phase 4, and pass the sidecar path to `evaluate()` explicitly.
7. **[S] The ≤2-loop reviewer-refinement pattern is quadruplicated.** `/diagram` (Phase 6 table), `/wireframes` (SKILL.md:406–430, "Hard cap: 2 loops per file"), `/prototype` (399–429, same), `/primer` (FR-8.3 auto-apply iteration) each restate loop budget, early-exit condition, and findings handling. Origin of the cap is principled — `2026-05-03_diagram-skill/02_spec.md` acceptance: "combined gate passes within at most 2 refinement loops on at least 4 of 5" — but the *protocol* (budget by rigor, early exit, findings → user → re-eval) belongs in `_shared/` (e.g., `_shared/reviewer-loop.md`) with per-skill parameters. Saves ~30 lines here and keeps the four skills' loops from drifting.
8. **[R] Path-convention incoherence.** Phase 0 `--selftest` runs `python3 skills/diagram/tests/run.py` and Phase 4 inlines `sys.path.insert(0, 'skills/diagram/tests')` "(Adjust path to wherever the skill repo lives)" — relative paths that break when installed as a plugin — while the comments-substrate section correctly uses `${CLAUDE_PLUGIN_ROOT}/skills/…`. **Fix:** use `${CLAUDE_PLUGIN_ROOT}` everywhere; drop the "adjust path" hedge.
9. **[P] Ceremony that costs words and trust.** style.md: "you MUST cite back to the relevant subsections — 'Using token `accent` per §5.1; legend per §5.6'" — chat citations enforce nothing the eval doesn't; delete. Phase 3 step 1: exact announce-string mandate ("Canvas: 16:10 because…") — "announce the canvas choice and why" suffices. Phase 2's verbatim AUQ YAML block could be one sentence of intent + the three option labels (the audit only needs the `(Recommended)`/defer-only tag, which is present).
10. **[V] File map (lines 447–491, 45 lines) is mostly `ls` output.** The load-bearing annotations are two: "atoms/ — NOT templates" and "editorial-v1.md — Phase 6b layout zones". Keep those two lines as prose pointers, delete the tree.
11. **[X] Cross-platform posture is genuinely good — keep.** Platform Adaptation gives concrete per-phase degradations (collision→suffix, brainstorm→first framing, findings→table); renderer detection has two non-Claude fallbacks with install hints; AUQ prose-fallbacks appear at each call site. The only soft spot is finding 1's subagent-context gap. Don't trade this away in trims.
12. **[V] Verbosity verdict: long for both reasons, in different sections.** The domain *is* complex (2 themes × 2 modes × deterministic eval × resolver contract × exit codes) — Phases 0, 5, 6a's exit table, and the resolver section earn their lines. The distrust-driven length is in Phase 3's token restatement, the anti-pattern list, the file map, and the AUQ verbatim blocks. ~526 → ~320 lines is achievable without touching any enforced contract.
