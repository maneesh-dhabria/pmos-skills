# frameworks — review

**Grade:** B (would be A− on design alone; an empirically confirmed matcher defect in the core retrieval path and body↔reference duplication hold it at B)
**Size:** SKILL.md 296 lines (212 excluding non-interactive block, lines 96–179); references 4 files / 427 lines; target ~170 lines (84 mandated block + ~85 body).

## TL;DR

- **Biggest win available:** fix the length-normalized confidence floor. A realistic verbose problem statement — exactly what the skill invites ("describe a problem in plain words") — scores 0.089, trips `low_confidence`, and is truncated to 2 candidates **before** the LLM re-rank can see the pool. Verified live: a CEO-pet-features prioritization query returns 2 low-confidence hits while "prioritize roadmap" returns a confident 5. The skill's core promise is "precise retrieval"; the scorer punishes the input style the description advertises.
- **Biggest risk in current design:** the deterministic-prefilter/LLM-re-rank split is duplicated three times (Phase 2, Anti-Patterns, `reference/matching.md`), so the next behavior change has three places to drift — this already happened once with the D5 `/diagram` correction, which was fanned out everywhere except the Anti-Patterns bullet, leaving SKILL.md contradicting itself.
- **Done well, worth keeping:** the architecture is the best in the learnkit plugin and shows the older skills' lessons learned — offline-first runtime over a shipped corpus, zero-dep Stage-A scripts each with `--selftest` (magazine's pattern), LLM derivation cached at sync-time never recomputed at runtime, a closed tag registry that makes matching auditable, ship-with-warning failure isolation, and genuine progressive disclosure ("the deep mechanics live one hop away in `reference/` — keep this body lean").

## Findings

1. **[G] Confidence floor falsely fires on verbose realistic queries — and truncates the pool before the LLM can rescue it.** `match.mjs:41` normalizes by `q.size × 3` (max-achievable if *every* token were a ×3 tag hit), so a 25-token natural problem statement where 4 tokens hit tags lands well under the 0.15 floor. `match()` then returns `scored.slice(0, 2)` (`match.mjs:52`), so Phase 2 step 3's LLM re-rank — the layer that "catches intent" per `reference/matching.md` — receives only 2 candidates instead of 15. Empirically confirmed against the shipped 272-record corpus: *"my CEO keeps asking me to add his pet features to the roadmap and I do not know how to push back and decide what we should actually build next quarter"* → `low_confidence: true, count: 2, top score 0.089`. The no-pad rule itself is right and origin-traceable (requirements doc: "'low confidence' rather than padding to 5", resolving OQ2 — `docs/pmos/features/2026-06-07_frameworks-skill/01_requirements.html`); the defect is the normalization + early truncation. **Fix:** normalize by something insensitive to query length (e.g. cap the denominator at ~6 effective tokens, or use best-k token hits), and on `low_confidence` still pass the top ~15 to the LLM re-rank on the human path — apply the ≤2 cap to the *output*, not the candidate pool. Add a verbose-query case to `--selftest`.
2. **[G] Pure-nonsense queries return 2 fabricated zero-score matches.** `match.mjs:52` slices the top 2 of `scored` (not `nonzero`) when low-confidence, so "zxqw flibber" returns `analytics/cohort-analysis` and `analytics/correlation-analysis` at `score: 0` with `why: "low-confidence match (name/context overlap)"` — verified live. That `why` is false (there is no overlap), which quietly violates the skill's own Anti-Pattern ("never fabricate"). **Fix:** filter to `score > 0` before the low-confidence slice; return `count: 0` for true nonsense. One-line change; `json-contract.test.mjs` only asserts `≤2`, so tighten it to assert 0-score exclusion.
3. **[R] SKILL.md contradicts itself about who generates diagrams.** Anti-Patterns line 274–275: "Diagrams are owned SVGs, **generated by `/diagram`**, inlined" — but Platform Adaptation (lines 47–48) and `reference/ingestion.md`'s mechanism note say the opposite: D5 was corrected at execute time because `/diagram` is structurally interactive and "cannot run unattended at corpus scale"; Stage-B agents generate SVGs directly. The correction was fanned out to two of three mentions. A future maintainer reading only the Anti-Patterns section will re-attempt the infeasible `/diagram` batch. **Fix:** reword the bullet to "owned SVGs generated at sync time, inlined".
4. **[V] Phases 2 and 5 re-explain their reference files instead of pointing at them.** Phase 2's 4-step matcher walkthrough (~22 lines) restates `matching.md` (weights, floor, no-pad, re-rank, --json prose suppression); Phase 5's 6-step pipeline (~28 lines) restates `ingestion.md`'s "Pipeline steps" nearly 1:1. This is the duplication that produced finding 3. The Pocock pattern the body itself claims ("keep this body lean") is to state the branch decision and the one rule deviation breaks, then defer. **Fix:** Phase 2 → ~8 lines (situation shortcut, `match.mjs` invocation, "LLM re-ranks and writes a ≤1-sentence why, cap 5; below-floor → ≤2 with caveat, never pad", present + open library). Phase 5 → ~6 lines (confirm prompt, "follow `reference/ingestion.md` end to end", the never-disturb-shipped-corpus guarantee). Saves ~35 lines and removes the drift surface.
5. **[G/F] The `--json` API is well-shaped but has three contract wobbles.** (a) **Non-determinism by context:** `matching.md` says the LLM re-rank runs on the `--json` path "when session context is available" and the prefilter answer stands otherwise — same call, different results, and the consumer can't tell which it got. The contract test only ever exercises the deterministic half. Add a `"reranked": true|false` field (or drop re-rank from `--json` entirely; determinism was the stated rationale for the prefilter). (b) **Unresolvable `diagram` paths:** `"data/diagrams/product__rice.svg"` is relative to `${CLAUDE_SKILL_DIR}`, which a consuming skill doesn't know — emit absolute paths or document the base. (c) **Null-convention inconsistency:** `category` defaults to `null` but `decision_type` to `"n/a"` (`match.mjs:75–76`). Minor, but contracts live on consistency. Also note: there are still **zero live consumers** ("wiring `/ideate` … is a `/backlog` follow-up" — `matching.md`), so these are cheap to fix now and expensive after.
6. **[R] `matching.md` shows an invocation that doesn't exist.** Line 63: `/frameworks --query "<problem>" --json` — the slash command takes a positional problem string (per `argument-hint` and Phase 1 dispatch); `--query` is `match.mjs`'s flag. The doc conflates the two surfaces. Same file says "an LLM ranking all ~200 records" — the corpus is 272. **Fix:** correct the invocation, say "~270".
7. **[F] `--format <html|md|both>` is a dead flag.** It sits in `argument-hint`, yet Phase 0 admits "v1 emits HTML only; `--format both` is reserved" — and ~6 lines of precedence-resolution + stderr-print ceremony exist to resolve a choice with one outcome. A reserved flag in the hint string is a promise the skill doesn't keep. **Fix:** drop the flag and the resolution paragraph until md output exists (the settings default already covers future needs).
8. **[Ph/V] "Track Progress" mandates per-phase tasks for phases that don't run sequentially.** Lines 55–61 say "multiple sequential phases. Create one task per phase" — but Phase 1 is a dispatcher and exactly one of Phases 2/3/4/5 runs per invocation; three of the "phases" are single tool calls. The section then half-retracts itself ("the runtime paths are short"). Boilerplate inherited from the pipeline skills. **Fix:** delete, or one sentence: "`sync` is long — track its 6 steps as tasks; runtime paths don't need tracking." Relatedly [Ph], Phase 6 (Capture Learnings) is numbered after the unnumbered Anti-Patterns section — ordering reads as accretion; move Anti-Patterns after Phase 6 or unnumber Phase 6 as "Before finishing".
9. **[S] Substrate use is correct; one global substrate bug surfaces here.** The skill properly inlines `_shared/pipeline-setup.md`, defers prompts to `_shared/interactive-prompts.md`, and carries the canonical non-interactive block. `build-library.mjs` owning its HTML (instead of `_shared/html-authoring/`) is justified: the library is an offline interactive app, not a commentable pipeline artifact — same precedent as `/magazine`'s `render-issue.js`, and it isn't one of the 14 pmos-comments surfaces. One cross-plugin note (assess globally, not a frameworks defect): the byte-identical block forces `pmos-toolkit: /<skill> finished` into the end-of-skill summary (`_shared/non-interactive.md:100`) — wrong plugin name for every learnkit skill.
10. **[P] The deterministic-prefilter raw ordering is rough, and the `--json` path may ship it raw.** "prioritize roadmap" tops `product-market-fit/superhuman-pmf-engine` (0.667) above the RICE-family frameworks — bag-of-words tag overlap, fine as a *prefilter*, but per finding 5a the programmatic path can emit it as the final answer with a templated `why`. Acceptable if consumers are told it's a prefilter; today `matching.md` oversells it as "an acceptable answer on its own."

### Special-attention verdicts

**Does the newest skill show the lessons of the older ones?** Largely yes — this is the most architecturally disciplined skill in learnkit. It inherits magazine's deterministic Stage-A/agentic Stage-B split, per-script selftests, closed tag registry, ship-with-warning isolation, and offline-first runtime; it inherits the substrate correctly; its body (212 effective lines) is leaner than primer (395) or magazine (320) and explicitly aspires to progressive disclosure. The repeated mistakes are stylistic, not architectural: phase-numbered ceremony around a simple dispatcher, body text that re-explains its own reference files (which already caused one contradiction), and flag accretion (`--format` reserved at birth).

**`--json` API design:** right instincts — deterministic core, tested shape, stable ordering, honest `low_confidence` signal, explicit "no chat prose" rule. Three contract wobbles to fix before the first consumer wires in: context-dependent re-rank without a marker, skill-relative diagram paths, and the `null`/`"n/a"` inconsistency (finding 5).

**Match-quality rules (2–5, "why it fits"):** the caps and the no-pad rule are the best part of the design — opinionated, origin-traceable, and enforced in code rather than prose. The implementation undermines them: length-normalization makes the advertised input style ("describe a problem in plain words") the worst-served input, and the low-confidence path both truncates the LLM's candidate pool and fabricates zero-score matches for nonsense (findings 1–2). Fix those two and the retrieval story matches the pitch.
