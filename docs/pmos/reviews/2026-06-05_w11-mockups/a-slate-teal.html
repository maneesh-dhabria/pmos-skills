Specification · Tier 3

Inline doc comments

Stakeholders annotate any PMOS-emitted HTML artifact in the browser. Comment threads persist as an inline JSON block inside the HTML itself — no sidecar files; the artifact is the single source of truth. This mockup shows Direction A — Slate & Teal applied to a representative spec page.

Context & goals

The current substrate is clean but generic: Tailwind-blue accent, system fonts, a near-invisible footer. A reader could not tell a PMOS doc from any other SaaS export. This direction trades that for an ownable teal, a grotesque display face, and a section signature that repeats ~50× per document.

Identity is governed by one substrate, so making it distinctive is an afternoon of design intent — not a re-architecture.

Functional requirements

FR-1 — Masthead wordmark

Every artifact toolbar carries the pmos· lockup top-left, set in the display face with a teal trailing dot. It is the single most recognizable element and appears on all 14 HTML surfaces.

FR-2 — Section signature

Each <h2> renders a 3px teal left-bar and an auto-incrementing §N counter. Cheap to render, compounds recognition across a long spec.
