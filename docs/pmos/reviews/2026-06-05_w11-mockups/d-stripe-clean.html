pmos 02 — Spec: Inline doc comments

Specification · Tier 3

Inline doc comments

Stakeholders annotate any PMOS-emitted HTML artifact in the browser. Comment threads persist as an inline JSON block inside the HTML itself — no sidecar files; the artifact is the single source of truth. This mockup shows Direction D — Stripe Clean applied to a representative spec page.

Context & goals

A clean, modern-SaaS feel: Stripe's signature blurple with a subtle gradient on the wordmark, Inter throughout, generous whitespace, soft shadows, pill buttons, and 10–12px rounded surfaces. The section signature is a tiny rotated gradient marker — restrained, never loud.

Clean is a choice, not an absence — the discipline is in the spacing and the single accent.

Functional requirements

FR-1 — Masthead wordmark

The toolbar carries pmos in Inter with a blurple→violet gradient. Crisp, contemporary, instantly SaaS-native. It appears on all 14 HTML surfaces.

FR-2 — Section signature

Each <h2> gets a small rotated gradient square marker. Quiet enough to repeat 50× without fatigue, distinct enough to read as a system.
