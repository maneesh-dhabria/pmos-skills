Specification · Tier 3

Inline doc comments

Stakeholders annotate any PMOS-emitted HTML artifact in the browser. Comment threads persist as an inline JSON block inside the HTML itself — no sidecar files; the artifact is the single source of truth. This mockup shows Direction C — Spec-Sheet Mono applied to a representative spec page.

Context & goals

The current substrate is clean but generic: Tailwind-blue accent, system fonts, a near-invisible footer. This direction leans into the spec/plan/diagram domain — a monospace display face for every heading and the wordmark, a burnt-orange signal accent, and bracketed [N] section tags as the repeated signature. It reads like a precise engineering spec-sheet.

A signature structural detail repeated ~50×/doc compounds recognition.

Functional requirements

FR-1 — Masthead wordmark

The toolbar carries pmos▍ in JetBrains Mono with a block-caret accent — unmistakably developer-native. It appears on all 14 HTML surfaces.

FR-2 — Section signature

Each <h2> is an uppercase mono header prefixed with an auto-incrementing [N] tag and underlined by a thin orange rule.
