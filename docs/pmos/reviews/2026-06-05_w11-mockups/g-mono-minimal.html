Specification · Tier 3

Inline doc comments

Stakeholders annotate any PMOS-emitted HTML artifact in the browser. Comment threads persist as an inline JSON block inside the HTML itself — no sidecar files; the artifact is the single source of truth. This mockup shows Direction G — Mono Minimal, a hybrid of C's monospace character and F's hairline restraint.

Context & goals

Keeps C's defining moves — JetBrains Mono headings, the burnt-orange signal accent, and the pmos▍ block-caret wordmark — but adopts F's discipline: hairline borders instead of heavy rules, generous whitespace, a black minimal button, and zero-padded section counters. The result is spec-sheet character with a calmer, more confident page.

Same voice as C, quieter delivery — the mono does the talking, the layout gets out of the way.

Functional requirements

FR-1 — Masthead wordmark

The toolbar carries pmos▍ in JetBrains Mono with a burnt-orange block caret — C's wordmark, unchanged. It appears on all 14 HTML surfaces.

FR-2 — Section signature

Each <h2> sits under a hairline rule (F), prefixed by a bracketed, zero-padded mono counter [01], [02]… in burnt-orange (C's brackets + F's zero-pad). Uppercase tracked headings keep the spec-sheet read.
