Specification · Tier 3

Inline doc comments

Stakeholders annotate any PMOS-emitted HTML artifact in the browser. Comment threads persist as an inline JSON block inside the HTML itself — no sidecar files; the artifact is the single source of truth. This mockup shows Direction B — Editorial Aubergine applied to a representative spec page.

Context & goals

The current substrate is clean but generic: Tailwind-blue accent, system fonts, a near-invisible footer. This direction reaches for a magazine-editorial feel — a high-contrast variable serif (Fraunces) for every heading and the wordmark, an ownable aubergine, and oversized section numerals as the repeated signature.

Use real typography — it is the primary lever for distinctiveness.

Functional requirements

FR-1 — Masthead wordmark

The toolbar carries pmos set in Fraunces italic, aubergine — a literary lockup that reads as a publication, not an export. It appears on all 14 HTML surfaces.

FR-2 — Section signature

Each <h2> is prefixed by an oversized serif §N numeral in aubergine. The numeral is the signature — no left bar needed; it carries the editorial character down the page.
