The problem

Great prompts don't compound.

You re-explain your process to the agent on every task. Quality drifts. There's no gate between "looks done" and "is done", and nothing carries forward to the next feature. A clever one-off prompt is not a repeatable pipeline.

How it works

A pipeline, not a pile of prompts.

Each stage is a skill with its own gate. The output of one is the input to the next, and the contract is enforced — so the agent runs the same disciplined process every time.

Shape & spec

Turn an idea into requirements and a reviewed spec — with grilling and creativity passes built in.

Plan & execute

Decompose into tasks, then build — worktree-isolated, test-driven, resumable.

Verify & ship

Binary skill-eval, hygiene lints, and a release train that bumps, tags, and pushes.

Is this for me?

Objections, handled.

Am I locked into one vendor?

No. Skills are plain Markdown under an MIT license, installed over the standard Claude Code marketplace mechanism. Fork any skill.

Will it bloat my setup?

Each plugin versions independently — install just the delivery pipeline, or add the learning, utility, manager, and game kits as you need them.

Is the "process" real, or just prompts?

Real. Every skill passes a binary skill-eval rubric, hygiene lints, and a non-interactive contract before it can ship — enforced in CI, not by vibes.

How do I get updates?

Updates flow through the marketplace; reinstall or update from the same source and your skills move to the latest gated release.
